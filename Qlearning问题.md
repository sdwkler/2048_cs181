# 2048 强化学习机制消融与分布式风险控制：完整架构演进记录

本记录详细阐述了在 2048 强化学习中，从基础 $Q(s,a)$ 到后状态（Afterstate）$V(s')$，再到均值-方差惩罚 $MV(s')$ 的五组消融实验的完整生命周期。

## 一、 核心状态流转与符号定义

在单步 MDP（马尔可夫决策过程）中，2048 的环境流转被严格拆分为三个物理节点：
* **$S$ (State)**：滑动前盘面（轮到玩家操作，等待输入动作 $a$）。
* **$S'$ (Afterstate)**：滑动后中间盘面（方块已滑动合并，产生即时得分 $R$，系统**尚未**生成随机方块）。
* **$S_{next}$ (Next State)**：下一回合盘面（系统已在 $S'$ 的空位随机生成 2 或 4，等待玩家下一次操作）。

---

## 二、 五大核心架构的单步流转与计算流程

以下流程展示了 AI 处于状态 $S$ 时，从“思考”到“执行”再到“复盘更新”的完整单步闭环。

### 🧪 实验 3-A：Q(s,a) + State (传统基线)
**设计理念**：教科书级的 Q-Learning，用 4 个动作脑死记硬背初始盘面。
1. **特征提取**：网络蒙上眼睛，只提取滑动前状态 $S$ 的特征。
2. **动作选择**：遍历合法动作 $a$，分别向 `Q_head[a]` 查询 $S$ 的得分，选择 $a^* = \arg\max_a Q_a(S)$。
3. **环境流转**：执行 $a^*$，获得奖励 $R$ 和下一状态 $S_{next}$。
4. **计算目标 (TD Target)**：在 $S_{next}$ 处，假想尝试所有动作，取出最高预测分：$Target = R + \gamma \max_{a'} Q_{a'}(S_{next})$。
5. **网络更新**：用 $Target$ 减去当前的 $Q_a(S)$ 得到 TD 误差，更新专属的 `Q_head[a]`。

### 🧪 实验 3-B：Q(s,a) + Afterstate (表征解耦)
**设计理念**：让网络看正确的中间盘面，但依然用 4 个动作脑去强行割裂经验。
1. **特征提取**：假想执行动作 $a$，提取滑完后的纯净后状态 $S'$ 的特征。
2. **动作选择**：向 `Q_head[a]` 查询 $S'$ 的得分，选择 $a^* = \arg\max_a Q_a(S')$。
3. **环境流转**：执行 $a^*$，获得奖励 $R$ 和下一状态 $S_{next}$。
4. **计算目标**：$Target = R + \gamma \max_{a'} Q_{a'}(S'_{next})$。
5. **网络更新**：更新专属的 `Q_head[a]`，但绑定的特征是 $S'$。

### 🧪 实验 3-C：V(s') + State (逻辑证伪组)
**设计理念**：反向证明组。用了全局统一大脑 V，但强制它去看滑动前的状态 $S$。
1. **特征提取**：提取滑动前状态 $S$ 的特征。
2. **动作选择**：核心 Bug 显现！评估任何动作 $a$ 时，公式为 $R_a + V(S)$。因为 $V(S)$ 是一样的，动作评估完全退化为贪心比较即时奖励 $R_a$。
3. **环境流转**：执行贪心动作 $a^*$，获得奖励 $R$ 和下一状态 $S_{next}$。
4. **计算目标**：$Target = \gamma \max_{a'} [R_{a'} + V(S_{next})]$。
5. **网络更新**：更新全局 `V_head`，绑定特征 $S$。

### 👑 实验 3-D：V(s') + Afterstate (完全体解耦)
**设计理念**：剥离环境随机性（看 $S'$）与剥离动作隔离性（用全局 V），完美经验共享。
1. **特征提取**：假想执行动作 $a$，提取后状态 $S'$ 的特征。
2. **动作选择**：向全局 `V_head` 查询 $S'$ 本身的好坏，选择 $a^* = \arg\max_a [R_a + V(S'_a)]$。
3. **环境流转**：执行 $a^*$，获得奖励 $R$ 和下一状态 $S_{next}$。
4. **计算目标**：$Target = \gamma \max_{a'} [R_{a'} + V(S'_{next})]$。
5. **网络更新**：计算 $Target - V(S')$ 得到 TD 误差，更新全局 `V_head`。

### 🛡️ 实验 3-E：MV(s') + Afterstate (均值-方差风险控制)
**设计理念**：在 3-D 的基础上，额外引入 `M_head` 学习平方期望，防范极端的随机暴毙风险。
1. **特征提取**：假想执行动作 $a$，提取后状态 $S'$ 的特征。
2. **动作选择（风险规避）**：向 `V_head` 查均值，向 `M_head` 查平方期望。
   * 计算方差：$Var(S'_a) = \max(0, M(S'_a) - V(S'_a)^2)$
   * 风险惩罚决策：$a^* = \arg\max_a [R_a + V(S'_a) - \lambda \sqrt{Var(S'_a)}]$
3. **环境流转**：执行 $a^*$，获得 $R$ 和 $S_{next}$。
4. **计算目标（双矩更新）**：
   * $Target_V = \gamma \max_{a'} [R_{a'} + V(S'_{next}) - \text{penalty}]$
   * $Target_M = R^2 + 2\gamma R \cdot V(S'_{next}) + \gamma^2 M(S'_{next})$
5. **网络更新**：分别计算 V 误差和 M 误差，同步更新 `V_head` 和 `M_head`。

## 三、 全局基石机制：TD(λ) 资格迹 (Eligibility Traces)

**核心声明**：$TD(\lambda)$ 并不是 3-E 的专属，它是所有 5 组实验、所有网络头（Q、V、M）共同运行的底层基石！

在 2048 中，得到 2048 方块的巨大奖励往往在几千步之后才发生。为了解决信用分配（Credit Assignment）问题，我们引入了资格迹：
1. **记忆刻痕**：每当访问一个盘面特征（如 $S'$）时，它在网络中的“活跃度（Trace）”加 1。
2. **时光衰减**：在每一步流转中，所有历史盘面的活跃度都会乘以衰减系数 $\gamma \cdot \lambda$。
3. **全局广播**：当我们在当前步计算出 $TD\_Error$ 时，我们不仅更新当前盘面，而是将这个误差**乘以所有历史特征的活跃度**，一次性追溯并更新过去几百步的盘面权重。

## 四、 均值-方差评估器 (MV) 的底层计算细节

为了在纯线性查表网络（N-Tuple）中实现分布式风险控制，我们在代码中进行了严密的数学降维与数值保护：

1. **缩放保护 (Scaling)**
   因为 2048 后期的得分极大（成千上万），直接计算平方期望 $M = \mathbb{E}[Return^2]$ 会导致浮点数溢出（Explosion）或梯度发散。我们在代码中强制施加了 `scale = 0.01`：
   $R_{scaled} = R \times 0.01$
   $V_{scaled} = V \times 0.01$
2. **方差截断 (Variance Clipping)**
   由于 V 和 M 分别由两个独立的 N-Tuple 头学习，在初期波动时，可能出现 $M < V^2$ 的非物理情况（导致方差为负）。因此决策公式严格保护为：
   `variance = max(0.0, m_val - v_val * v_val)`
3. **学习率解耦**
   为了防止 M 头的震荡干扰 V 头的收敛，我们为 `M_head` 设置了更小、更稳健的学习率（`alpha_M = alpha_V * 0.1`）。
