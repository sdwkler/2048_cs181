## 一、研究概览与核心命题

本项目围绕 Afterstate 解耦机制，在 Expectimax、MCTS、TD 学习三类算法框架下，分三个阶段递进验证。每个阶段基于前一阶段暴露的问题，提出更深入的实验假设。

```
Phase 1（浅层，§6.1–§6.4）—— 标准环境下的基准消融
  ├── 固定 P(4)=10%，控制变量法剥离 Afterstate 的树结构与评估对象两个维度
  ├── 在 Expectimax、MCTS（纯人工价值）、TD 学习（25K + 100K）三类方法下
  └── 量化证明 Afterstate 的联合解耦优势，但发现错配灾难（1-E：−92%）和过估计结构性回归

Phase 2（中层，§6.5）—— 环境漂移鲁棒性测试
  ├── 人为注入 P(4) 从 10%→90% 的环境偏移，检验「解耦环境 ≠ 免疫环境」
  ├── Expectimax + N-Tuple 漂移（发现 Afterstate 鲁棒性弱于 State）
  ├── MCTS + 人工价值漂移（两者同步衰减，Afterstate 高偏移时被反超）
  └── Q-Learning 零样本迁移（全面崩溃至 <6K）

Phase 3（深层，§6.6）—— 架构演进与深度优化
  ├── Q-Learning：TDA-Full 精确期望 + Downside-MV 双头方差（未超越简单采样版）
  ├── Expectimax：HashDAG（无损）+ BeamSearch（深度依赖）
  └── MCTS：接入 N-Tuple，追踪「State 矮胖 vs Afterstate 瘦深」的拓扑差异
```

本文整合 Phase 1–3 全部已完成实验的数据与分析。Phase 2 的梯度谱分析（方向二）及 POMDP 战争迷雾实验未执行，不纳入本文范围。

## 二、理论基础：Afterstate 解耦机制

### 2.1 2048 单步 MDP 的三节点模型

在 2048 的每一步流转中，存在三个严格区分的状态节点：

```
 S (State)           S' (Afterstate)        S_next (Next State)
┌─────────┐         ┌─────────┐            ┌─────────┐
│ 滑动前   │  玩家   │ 滑动后   │  环境随机  │ 下一轮   │
│ 完整盘面 │ ─────→ │ 中间盘面 │ ────────→ │ 完整盘面 │
│ 等待操作 │  动作a  │ 方块已合 │  生成2/4  │ 等待操作 │
│         │  得分R  │ 尚未生成 │            │         │
└─────────┘         └─────────┘            └─────────┘
```

- **S（State/决策节点）**：方块已生成完毕，轮到玩家滑动的完整盘面
- **S'（Afterstate/机会节点）**：玩家刚滑完、方块已合并、但**环境尚未随机生成新方块**的纯中间态
- **S_next（Next State）**：系统在 S' 的空位上随机生成了 2 或 4 后，进入下一个玩家回合

### 2.2 为什么 Afterstate 是一个解耦机制？

标准的 Q-Learning 直接从 S 到 S_next，将**玩家的确定性动作**和**环境的随机响应**糅合在一起。而 Afterstate 通过显式插入一个 **S' 节点**，将这两者强行分离：

- **S → S'**：纯确定性过程（玩家滑动），完全可控
- **S' → S_next**：纯随机性过程（环境生成方块），完全不可控

这种"解耦"的本质效果是：

1. **拓扑折叠**：多个不同的动作路径可能抵达同一个 S'（因为方块还没生成），搜索树从"指数爆炸的树"坍缩为"有向无环图（DAG）"
2. **方差缩减**：评估对象是 S'（纯净盘面），而非 S_next（已包含随机噪声的盘面），极大降低了采样方差
3. **目标纯净**：Bellman 更新不再需要 `max` 操作去处理环境随机性，而是直接评估 S' 本身的价值

## 三、研究设计

第二章从理论上论证了 Afterstate 在三个维度上具备优势。本章回答紧随其后的方法论问题：**面对这三个维度的理论声称，如何设计一套实验体系对其进行独立、可量化的验证？**

### 3.1 从理论维度到实验方法的映射

三个理论维度在性质上截然不同，单一实验方法无法同时覆盖：

| 理论维度 | 核心问题 | 需要的测量手段 | 对应实验 |
| --- | --- | --- | --- |
| 拓扑折叠 | 搜索树是否真的变瘦了？ | 精确计数树节点、计算分支因子 | **Expectimax** |
| 方差缩减 | 采样噪声是否真的降低了？ | 重复采样、统计估值分布 | **MCTS** |
| 梯度稳定 | 权重更新是否真的更平滑？ | 逐步追踪 TD 误差、测量收敛 | **TD 学习** |

三种方法之所以恰好构成互补，在于它们对"评估器"的依赖关系根本不同：

- **Expectimax 相信评估器**：它使用预训练的冻结权重作为完美的叶子节点裁判，穷举展开搜索树。正因如此，树结构的任何差异——节点是否被合并、分支因子是否缩减——会直接、干净地反映在决策效率和节点统计量上。它测的是**树长什么样**。
- **MCTS 不相信评估器（Phase 1）**：它使用纯手工价值函数（FastHeuristic）驱动的即时随机 Rollout 来估计动作价值。正因如此，采样过程的噪声水平——估值的方差、多次运行的一致性——会直接暴露在根节点统计量上。它测的是**采样有多吵**。Phase 3 进一步接入预训练 N-Tuple 估值，测试评估精度提升后树拓扑的形态变化。
- **TD 学习自己成为评估器**：它从零开始在线学习，权重每步都在更新。正因如此，更新信号的质量——TD 误差的波动幅度、过估计的膨胀程度——会完整记录在训练曲线上。它测的是**梯度往哪走**。

三种方法不是"都试试看哪个好"，而是**同一命题在三个维度上的独立证伪测试**——任何一个维度上的负面结果都会对 Afterstate 的理论完整性构成挑战。

### 3.2 Afterstate 的两个维度

在具体实验设计中，"Afterstate"并非一个不可分割的整体。它实际上包含**两个独立可操作、可独立切换的维度**：

**维度一：树结构的 Afterstate。** 控制搜索树（或规划树）是否在动作节点和随机落子之间显式插入 S' 机会节点。分离后，不同动作路径可能命中同一个 S'，树从"每路径唯一节点"变为"多路径共享节点"的 DAG 结构。这个维度回答的是：**树拓扑本身变瘦了，能带来多少收益？**

**维度二：评估对象的 Afterstate。** 控制打分器（无论是启发式函数还是 N-Tuple 网络）的输入是什么——是直接评估纯净的 S'（不含随机噪声），还是评估已经包含了随机落子的 S_next（含噪声）。这个维度回答的是：**看干净的东西打分，比看带噪声的东西打分，能带来多少收益？**

两个维度的独立性意味着可以构造 **2×2 消融矩阵**：树结构分离与否 × 评估对象分离与否。如果某一维度切换后得分不变，说明该维度在给定条件下不贡献增益；如果同时切换后得分大幅跃升，说明两个维度存在协同效应。这一矩阵逻辑贯穿全部三组实验的设计。

### 3.3 控制变量法的严格实施

所有实验共享一套严格的控制变量协议，确保观测到的任何差异唯一归因于 Afterstate 机制本身：

- **统一随机种子（seed=181）**：所有实验组从同一组种子派生，方块生成序列完全一致，排除"运气波动"的替代解释
- **双模式验证**：`--mode smoke`（1~3 局快速验证，确保代码正确）和 `--mode full`（完整统计，100 局搜索、10 局 MCTS、Phase 1 TD 学习 25,000 局 / 扩展及 Phase 3 100,000 局）保证迭代速度与统计可靠性兼顾
- **同一套底层系统**：三组实验共享完全相同的棋盘引擎和 N-Tuple 表征引擎，排除"差异来自实现细节"的可能性

## 四、实验平台

三组实验的独立验证需要一套共享的底层基础设施。本章描述这套平台的三个组成部分：棋盘引擎（定义游戏语义）、表征与评估（将棋盘转换为数值信号）、公共实验工具（将决策结果转化为可比较的证据）。

### 4.1 棋盘引擎：Bitboard 游戏模拟

#### 4.1.1 设计目标与核心策略

棋盘引擎是整个系统的基础。在 Expectimax 展开搜索树时，每步决策可能需要评估数千乃至数万个节点；在 MCTS 中，单局游戏可能执行数十万次 Rollout 滑动；在 TD 学习中，训练十万局意味着数千万次棋盘操作。如果单次滑动耗时在毫秒量级，深度搜索将不可接受。因此，棋盘引擎的设计目标明确：**将单次滑动（含方块合并、得分计算）降低至微秒量级**。

实现这一目标的核心策略是 **Bitboard 编码**——将 4×4 棋盘压缩进一个 64 位整数，利用 CPU 的位运算指令实现并行的行操作。这一思路源自国际象棋引擎的经典实践，在 2048 的简化棋盘上发挥得更为极致。

#### 4.1.2 核心数据结构：4-bit 编码

2048 棋盘的每一格取值均为 2 的幂：空（0）、2、4、8、…、32768，共计 16 种状态。这恰好可用 4 个二进制位编码（$2^4 = 16$），存储的是**指数**而非原值：

| 存储值（4 bit） | 对应方块值 |
| :-------------: | :--------: |
|    0 (0000)     |     空     |
|    1 (0001)     |     2      |
|    2 (0010)     |     4      |
|        …        |     …      |
|    15 (1111)    |   32768    |

棋盘共 4×4=16 格，每格 4 bit，总计 16×4=64 bit，恰为一个 64 位整数的宽度。采用**小端序**（低位在前）布局，棋盘左上角对应最低 4 位，右下角对应最高 4 位：

```
棋盘坐标                     raw 整数中的位布局
┌────┬────┬────┬────┐        bit[63]················bit[0]
│ 0  │ 1  │ 2  │ 3  │        高位                   低位
├────┼────┼────┼────┤        索引15 ← ··· ← 索引0
│ 4  │ 5  │ 6  │ 7  │        每格占连续4 bit
├────┼────┼────┼────┤
│ 8  │ 9  │10  │11  │        例如：raw = 0x0000_0000_0000_0321
├────┼────┼────┼────┤        从低到高：索引0=1(2)，索引1=2(4)，
│12  │13  │14  │15  │        索引2=3(8)，其余为0
└────┴────┴────┴────┘
```

> 选择 4 bit 而非 3 bit（不足）或 5 bit（浪费）并非为了节约内存（64 位早已是 CPU 原生宽度），而是为了获得天然的对齐优势：棋盘的一行恰好是连续的 16 bit。

#### 4.1.3 滑动计算：查表法实现 O(1) 移动

Bitboard 编码的直接红利是查表法。棋盘的一行包含 4 格 × 4 bit = 16 bit，其全部可能取值仅有 $2^{16}=65536$ 种。程序启动时，穷举 0~65535 所有行，对每一行预先模拟左滑（方块合并 + 靠拢），计算结果存入查表。此后运行时不再需要任何合并逻辑——拿到一行，用它的 16 bit 值当下标直接查表即可。

运行时：提取行 → 查表 → 拼接，每行 O(1)，4 行总计 O(1)。所有的合并决策逻辑（相邻是否相等、空格如何靠拢、得分如何计算）在查表预计算阶段就已执行完毕，运行时被压缩为一条内存读取。

#### 4.1.4 方向映射：位运算旋转统一四向

棋盘有四个滑动方向，但不需要为每个方向构建独立的查表。代码中只有一张查表（`lookup.find`），每个条目同时存储左滑（`.left`）和右滑（`.right`）结果——右滑结果本身通过镜像 → 左滑 → 镜像回来计算。上下方向则通过棋盘旋转转化为左右方向（`rotate_clockwise` = `transpose` + `mirror`，`rotate_counterclockwise` = `transpose` + `flip`）：

| 方向 | 等价操作                                                         |
| :--: | ---------------------------------------------------------------- |
| 左移 | 逐行查 `.left`                                                   |
| 右移 | 逐行查 `.right`（内部含镜像逻辑）                                |
| 上移 | `rotate_clockwise` → 逐行查 `.right` → `rotate_counterclockwise` |
| 下移 | `rotate_clockwise` → 逐行查 `.left` → `rotate_counterclockwise`  |

#### 4.1.5 完整滑动流程

以一次左滑（`move_left`）为例，将前述三个机制串联起来：

1. **保存原始盘面**——记录滑动前的 `raw` 值，用于最后判断滑动是否有效（棋盘是否发生了变化）。
2. **逐行查表**——对四行依次提取 16 bit 行值，以该值为下标直接查 `lookup.find`，取得预计算的左滑结果（新的行值 + 该行合并得分），将新行值按位 OR 累加到输出棋盘，将得分累加到总分。
3. **判断有效性**——若滑动后的棋盘与原始盘面完全相同，说明该方向无可合并的相邻方块且无空格可移入，返回 -1 表示非法动作。

> 上下方向的滑动流程完全相同，仅在查表前增加一步旋转（将上下方向映射为左右方向），查表后再旋转回来。整条流水线不包含任何逐格遍历的 for 循环和合并判断的条件分支——所有这些逻辑已在查表的预计算阶段执行完毕，运行时仅剩整数位运算和内存读取。

### 4.2 从棋盘到信号：表征与评估

棋盘引擎提供了游戏规则层面的状态表示——一个 64 位整数编码的完整盘面。但 AI 决策需要的不是原始状态本身，而是一个数值化的判断：这个盘面有多好？表征与评估层连接了底层游戏和上层决策AI。

三组实验对这一环节的依赖方式各不相同：

- Expectimax 需要冻结的预训练权重作为精确裁判；
- TD 学习需要从零开始在线更新的估值函数；
- MCTS 则完全绕开这一环节、靠即时随机模拟来替代估值。

#### 4.2.1 评估器总览

系统提供了四类评估器，覆盖从人工预设到数据驱动的完整谱系：

| 评估器 | 评分依据 | 权重来源 | 主要使用场景 |
| --- | --- | --- | --- |
| `HeuristicEvaluator` | 空位数 + 单调性 + 平滑度 + 角落最大值 | 人工经验预设 | Expectimax 无学习基线 |
| `EvolutionaryEvaluator` | 在启发式基础上增加蛇形链、最高方块比、可合并性 | 进化算法搜索 | Expectimax 进阶对照 |
| `NTupleEvaluator` | 封装预训练的 N-Tuple 网络，直接调用 `tdl.estimate(b)` | TD(0) 离线训练 | Expectimax 学习上限、TD 学习在线训练 |
| `FastHeuristic` | 与 HeuristicEvaluator 同一套评估逻辑 | 人工经验预设 | MCTS Rollout 专用（65536 行查表 + 位运算，O(1)，不涉及 N-Tuple） |

其中 `HeuristicEvaluator`、`EvolutionaryEvaluator` 和 `FastHeuristic` 不依赖任何预训练，权重由人工经验或进化算法直接给出。`NTupleEvaluator` 则需要离线预训练——它在TD learning 中起到核心作用。

各评估器在具体实验中的选择理由和绑定方式详见第五章各组实验方案。

> **评估器的双重角色。** 在运行时，评估器是决策的裁判——给搜索树的叶子节点或 MCTS 的 Rollout 终点打分以辅助动作选择，属于模型/策略的一部分。在实验结束后，评估器的估值与真实回报之间的偏差（如 Q-Learning 中的过估计偏差）本身成为分析对象。在 Expectimax 实验中，`NTupleEvaluator` 封装的 N-Tuple 权重是冻结的预训练快照（只读）；在 TD 学习实验中，同一 N-Tuple 的权重随 TD 更新持续变化（在线更新）——接口相同，角色不同。实验结束后对最终得分、达标率的汇总统计由 §4.3 的 `summarize_games` 完成，与评估器层的运行时打分是两条独立的通路。

#### 4.2.2 N-Tuple 网络：学习驱动的表征引擎

2048 棋盘的状态空间极为庞大——16 格 × 每格 16 种取值，理论空间即达 $16^{16} \approx 1.8 \times 10^{19}$。穷举存储不可行，手工设计评估函数则过于粗糙。系统需要一个轻量且可高效训练的函数逼近器：**给定棋盘，输出估值。**

N-Tuple 网络是 2048 AI 中经过验证的经典选择，相比深度神经网络有三个优势：

- 线性可训（TD(0) 在线更新，无反向传播）；
- 内存可控（权重表大小由 tuple 选取方案显式决定）；
- 旋转不变性通过同构映射天然嵌入而非靠数据学习。

##### 4.2.2.1 feature 底层机制

N-Tuple 与棋盘引擎的查表法完全同构：**将局部编码为整数索引，直接查数组取值**——区别在于棋盘引擎查的是"滑动后变成什么"（确定性预计算），N-Tuple 查的是"这个局部配置前景好不好"（从数据中学习）。

一个 `feature` 的具体工作方式：

- 在 16 格棋盘上选定 6 个固定位置，将 6 格的值逐位移拼接为 24 bit 索引（$2^{24} = 16\text{M}$ 条目），从权重表中查出该配置的估值。
- 同时，为消除方向偏见，每个 feature 配有 8 组不同位置的观察窗口——形状相同但朝向不同（4 个旋转方向 × 2 种镜像），分别覆盖棋盘的不同角落。
- 8 组窗口各自查表，结果全部加在一起作为该 feature 的估值，保证了同一配置无论出现在棋盘的哪个方向贡献相同。

##### 4.2.2.2 feature 构造方式

本项目构造 feature 采用 **绝对值 + 差分联合方案**，共 4 种 shape × 2 种编码 = **8 个 feature**：

- **4 个 `pattern`**：直接将 6 格的方块指数（0~15，4 bit/格）拼接为 24 bit 索引 → 权重表 $2^{24}=16\text{M}$。
- **4 个 `diff_pattern`**：不存方块值本身，存相邻位置的**差值**。相邻差的范围是 -15~+15（31 种可能，需 5 bit），加偏移 +15 映射到非负数后拼接为 25 bit 索引 → 权重表 $2^{25}\approx 33.5\text{M}$。

> 虽然`diff_pattern`只存 5 个差（比 6 个绝对值少一个数），但每个差 5 bit > 每个绝对值 4 bit，所以表反而更大。

两种编码方式互补：`pattern` 直接存储每格的绝对指数，见过的盘面估值精确；`diff_pattern` 存储相邻格的差值，形状相同而整体数值平移的盘面（如盘面上所有方块各升一级）编码完全一致、共享权重，从而将经验泛化到训练中从未出现的配置。一方保证精度，一方负责泛化，联合使用时估值既稳定又能覆盖未见局面。

##### 4.2.2.3 单步估值完整流程

对任意输入盘面，N-Tuple 网络的估值过程分为三层嵌套求和，由内向外依次为：

1. **窗口层（同构映射）**：每个 feature 的 8 组同构窗口各自查表取值，累加得到该 feature 的输出（详见 §4.2.2.1）。
2. **编码层（绝对值 + 差分）**：系统共包含 4 个绝对值 feature（`pattern`，直接编码方块指数）和 4 个差分 feature（`diff_pattern`，编码相邻格差值）。每个 feature 独立完成窗口层的求和，产生一个标量分值。
3. **汇总层**：8 个 feature 的分值相加，得到最终的盘面估值 V(S')。

一次完整的估值过程仅需访问 8 × 8 = 64 个浮点数。权重更新时，TD 误差均分到这 64 个被激活的权重上，每个权重各加 α × error / 64。

##### 4.2.2.4 训练与权重加载

N-Tuple 网络的权重在两组实验中对应两种不同的使用方式：

**离线预训练（Expectimax，§5.1）。** 由 `train_ntuple.py` 以 S 和 S' 为不同输入对象各训练 50,000 局，产出两套冻结权重：`2048_state.bin`（训练时观察含随机噪声的完整盘面 S）和 `2048_afterstate.bin`（训练时观察纯净盘面 S'）。两套权重由完全相同的网络结构和训练算法产生，唯一差异在于训练时的观测对象。这一设计为 §3.2 所述 2×2 消融矩阵中的"评估对象维度"提供了物理载体——加载 `2048_state.bin` 对应评估含噪声的 S，加载 `2048_afterstate.bin` 对应评估纯净的 S'，其余条件全部相同。

**在线训练（TD 学习，§5.3）。** 权重从零初始化，采用 TD(0) 算法在每一局结束后回溯更新：`target = R + V(s')` 作为一步 Bellman 目标，TD 误差乘以学习率后均分到激活的 64 个权重上。训练初期大量棋盘配置从未出现，使用稀疏字典按需存储，而非预分配完整权重数组。

## 五、实验方案

本章逐一展开三组实验方案，每组遵循统一结构：算法原理及 Afterstate 在此实验中的具体实现方式 → 评估器选择理由 → 实验矩阵与参数配置 → 专属评价指标及其理论动机。

### 5.1 Expectimax

Expectimax 是完美信息下的最优搜索策略——假设评估器给出的叶子节点估值完全准确，则深度优先穷举所有可能的动作序列和随机结果，取期望最大的动作。在 2048 中，Expectimax 的搜索树天然包含两类节点：决策节点（玩家选择动作）和机会节点（环境随机落子）。Afterstate 的介入改变了这两类节点的连接方式。

**核心目的：** 探究在人类先验知识（Heuristic）与 AI 自主学习（N-Tuple）两种打分器下，Afterstate 树结构究竟是如何改变搜索效率和决策质量的。

#### 5.1.1 算法与 Afterstate 实现

搜索树的结构对比：

```
【Standard Expectimax（不分离）】          【Afterstate Expectimax（分离）】
       S (决策节点)                              S (决策节点)
      / |  \                                  / | \
    a0 a1 a2 a3                             a0 a1 a2 a3
     |  |   |  |                              |  |  |  |
    [环境随机生成]                           S'0 S'1 S'2 S'3 (机会节点)
     |  |   |  |                              |  |  |  |
    S0 S1  S2  S3  (下一层决策节点)          [环境随机生成]
                                               |  |  |  |
                                              S0 S1 S2 S3

  ❌ 每个动作下挂N个随机子节点              ✅ S' 节点充当"拓扑漏斗"
  ❌ 两路径到同一S_next概率极低               ✅ 多路径高频命中同一S'
  ❌ N_unique ≈ N_total                      ✅ N_unique << N_total (压缩率→0.5)
```

在 Standard 模式下，每个动作 a_i 之后，环境立即在所有空位上随机生成新方块，产生 N 个不同的 S_next。搜索树在机会节点处爆炸式分岔。在 Afterstate 模式下，动作 a_i 之后首先抵达一个共同的 S' 节点（仅含合并结果），然后才在 S' 的基础上分岔到不同的 S_next。由于不同动作可能产生相同或相似的 S'（例如左滑和右滑在对称盘面上产生镜像结果），搜索树从"每路径独立展开"的树状结构坍缩为"多路径共享 S'"的 DAG 结构。

在 Expectimax 实验中，"Afterstate"实际指代 §3.2 所述的两个独立可拆分的维度，本组实验的 2×2 控制变量矩阵正是为了将它们拆开：

**树结构维度（`use_afterstate`）**：控制搜索树是否在动作节点和随机落子之间插入 S' 机会节点

**评估对象维度（`leaf_mode` + `eval_type`）**：控制两件事：

- **(1) 叶子节点给什么盘面打分:** 是直接评估纯净的 S'（`leaf_mode="afterstate"`），还是模拟环境随机落子后评估含噪声的 S_next（`leaf_mode="state"`）；
- **(2) 打分器本身是用什么数据训练的:** `2048_afterstate.bin`（训练时看 S'）还是 `2048_state.bin`（训练时看 S_next）

#### 5.1.2 评估器选择与实验矩阵

本组实验使用三类评估器：

- **HeuristicEvaluator**：作为无学习基线。评估指标为空格数、单调性、平滑度、角落最大值。权重由人工经验预设（`w_empty=270, w_mono=47, w_smooth=0.1, w_corner=500`）。在矩阵中标记为"看 S'"或"看 S_next"，取决于叶子节点评估时传入的 `is_afterstate` 参数。
- **NTupleEvaluator（State 版）**：加载 `2048_state.bin`，训练时输入为含随机噪声的完整盘面 S。在 Standard 树中评估 S_next 叶子，在 Afterstate 树中评估 S 决策节点。
- **NTupleEvaluator（Afterstate 版）**：加载 `2048_afterstate.bin`，训练时输入为纯净盘面 S'。在各树结构中都评估 S' 叶子——这是"完全解耦"的配置。

> 在消融矩阵中，1-D（Standard + State NT + leaf_mode=state）和 1-G（Afterstate + Afterstate NT + leaf_mode=afterstate）是两组"正确匹配"的配置——打分器训练分布与叶子评估对象一致。1-E（Standard + Afterstate NT + leaf_mode=state）和 1-F（Afterstate + State NT + leaf_mode=afterstate）是两组"错配"配置——打分器训练分布与叶子评估对象不一致。消融实验正是要通过这四组的对比，量化两种错配各自的代价和不对称性。

具体实验如下：

| 实验 | 树结构 | 打分器 | 类别 |
| --- | --- | --- | --- |
| 1-A | Greedy（深度=1） | Heuristic（看 S'） | 绝对基线 |
| 1-B | Standard Expectimax（不分离） | Heuristic（看 S_next 叶子） | 普通搜索 |
| 1-C | Afterstate Expectimax（分离） | Heuristic（看 S' 叶子） | 架构解耦 |
| 1-D | Standard（不分离） | State N-Tuple（看 S） | 标准传统 |
| 1-E | Standard（不分离） | Afterstate N-Tuple（看 S'） | **表征错配**——树不分离+Afterstate打分器，喂S_next给训练看S'的打分器 |
| 1-F | Afterstate（分离） | State N-Tuple（看 S） | 架构解耦——树分离+State打分器，喂S'给训练看S的打分器 |
| 1-G | Afterstate（分离） | Afterstate N-Tuple（看 S'） | **完全解耦** |
| 1-H | Afterstate + Afterstate NT + 剪枝 | 同 1-G + 前向剪枝 | 终极拓展 |

#### 5.1.4 专属评价指标

除 §4.3.2 定义的通用指标外，Expectimax 另有三项特殊指标，用于精确量化 Afterstate 对树拓扑的影响：

**有效分支因子（Effective Branching Factor）**：

$$b_{\text{eff}} = N_{\text{nodes}}^{1/d}$$

其中 $N_{\text{nodes}}$ 为整棵搜索树中展开的节点总数，$d$ 为搜索深度。$b_{\text{eff}}$ 度量的是"平均每加深一层，树变胖多少倍"。树越瘦（节点去重越充分），$b_{\text{eff}}$ 越低。

**压缩率（Compression Ratio）**：

$$\text{Compression Ratio} = \frac{N_{\text{unique}}}{N_{\text{total}}}$$

其中 $N_{\text{unique}}$ 为去重后的唯一状态数，$N_{\text{total}}$ 为搜索过程中访问状态的总次数。压缩率直接量化"有多少节点是重复展开的"。  
压缩率越低，说明 DAG 折叠效果越显著。架构不分离时压缩率 $\to 1.0$（几乎无折叠）；架构分离后压缩率 $\to 0.5 \sim 0.55$（约一半节点被去重）。

> $b_{\text{eff}}$ 和压缩率本质上是同一件事的两种度量——Afterstate 让不同路径合并到同一个 S'，搜索树变瘦。$b_{\text{eff}}$ 量的是后果（平均每层分岔数），压缩率直接数重复（多少节点白展开了）。

**决策分歧遗憾值（Action Disagreement Regret）**：

在 500 个高压盘面（空格 $\leq 3$）上，分别让 1-D（Standard + State N-Tuple）和 1-G（Afterstate + Afterstate N-Tuple）各自决策。若两算法推荐的动作不同，以完全解耦的 1-G 为基准，计算未解耦算法选错动作的代价：

$$\text{分歧率} = \frac{\text{决策不同的盘面数}}{\text{总盘面数}}$$

$$\text{遗憾值} = V_{\text{1G}}(A_{\text{1G}}) - V_{\text{1G}}(A_{\text{1D}})$$

该指标回答一个关键问题：Afterstate 的"估值更准"是否真的转化为"决策更好"，以及决策分歧在高压局面下有多大的实际代价。

### 5.2 MCTS

MCTS（蒙特卡洛树搜索）与 Expectimax 的根本不同在于：它不依赖任何预训练的评估器来给出叶子节点的精确估值，而是靠**当场随机模拟**（Rollout）来判断动作好坏——从当前盘面出发，对每个可行动作重复执行"随机滑一步 → 环境落子 → 再随机滑……"直到终局或步数上限，记录累计得分，取多次 Rollout 的平均分作为该动作的估值。

**核心目的：** 在**不使用**任何离线预训练 N-Tuple 模型的前提下，纯粹依赖即时随机 Rollout，探究 Afterstate 树结构对降低采样方差的真实贡献。

#### 5.2.1 算法与 Afterstate 实现

MCTS 的搜索树也由决策节点和机会节点交替构成，与 Expectimax 同构。Afterstate 的区别体现在**随机噪声被固定的位置**：

- **Standard MCTS**：滑一下，系统立刻生成新方块，这个随机方块成为棋盘的一部分被固定下来，然后 Rollout 从这个已含噪声的棋盘起步。200 次 Rollout 的起点各不相同（方块掉在不同空格），噪声变成了"事实"。
- **Afterstate MCTS**：滑一下，趁新方块还没生成，直接从这个干净的合并后盘面起步 Rollout。新方块不在起步时出现，而是被推进 Rollout 内部——成为 Rollout 里每一次滑行之后自然发生的随机事件。

后果：Standard 的 200 次 Rollout 起步点各不相同，噪声直接污染了估值基线；Afterstate 的 200 次 Rollout 都从同一个干净盘面起步，噪声被推入后续成千上万次随机采样中，通过取平均消解掉了。这就是"方差缩减"的 MCTS 版本。

#### 5.2.2 Rollout 策略与 FastHeuristic

本组实验**完全不使用**任何预训练 N-Tuple 模型。每次 Rollout 的滑行策略为：90% 概率走启发式贪心（用 FastHeuristic 给四个方向快速打分，选最高的），10% 概率真随机。

FastHeuristic 是一个手工设计的轻量级评估器，评估指标与 `HeuristicEvaluator` 相同（空格数、单调性、平滑度、角落最大值），区别在于所有 16 格的 for 循环全部换成 65536 行查表 + 纯位运算，不放缓存也能 O(1)，支撑 Rollout 每步数千次的调用。

每次 Rollout 有步数上限（`rollout_limit=5`），到达上限后用 FastHeuristic 对当前盘面打分作为终局估值代理。每步决策时，四个方向的 Rollout 预算（200/500/1000/2000）是共享的，由 PUCT 公式自动分配：

$$\text{PUCT} = V_{\text{score}} + c_{\text{pb}} \cdot \frac{\sqrt{N_{\text{parent}}}}{N_{\text{child}} + 1}$$

其中 $c_{\text{pb}} = \log\left(\frac{N_{\text{parent}} + C_{\text{base}} + 1}{C_{\text{base}}}\right) + c_{\text{init}}$，对齐 Stochastic MuZero 的探索调度策略。得分高的方向多试，得分少的少试。初始时四个方向估值相同，PUCT 的探索项迫使每个方向都先试几次，在尝试中发现哪个方向好，后续自然倾斜过去。

#### 5.2.3 实验矩阵与参数配置

- **稳定性测试**——每个盘面重复独立运行 50 次，统计首选动作分布；
- **整局游戏测试**——跑 10 局完整游戏，记录得分、达标率和耗时。

所有实验共享随机种子（seed=181），且同一 Rollout 档位下 Standard 和 Afterstate 的 MCTS agent 使用**完全相同的 RNG 初始状态**，Rollout 随机序列完全一致，确保两组之间的唯一差异是树结构。

#### 5.2.4 专属评价指标

**根节点动作估值方差（根方差，Root Action-Value Variance）**：

MCTS 搜索结束后，计算根节点四个动作的 Q 值标准差：

$$\sigma_{\text{root}} = \text{std}(Q_{\text{up}}, Q_{\text{right}}, Q_{\text{down}}, Q_{\text{left}})$$

方差大： AI 能清晰区分动作优劣，决策有依据。  
方差 $\to 0$ ： 四个方向估值趋同，AI 在噪声中"瞎蒙"。

拥挤盘面下，环境随机落子噪声大，普通 MCTS 算出的四个方向估值极其接近（方差小），而 Afterstate MCTS 预期能拉开差距。该指标直接反映算法对动作优劣的"辨别清晰度"。

**策略熵（Policy Entropy）**——反映决策稳定性：

同一盘面重复独立运行 50 次，统计最佳动作分布，计算策略熵：

$$H(\pi) = -\sum_{a} p(a) \log p(a)$$

策略熵低： 50 次全选同一个方向，决策稳定。  
策略熵高： 50 次选了不同方向，算法被 Rollout 噪声左右。

该指标直接量化了"Afterstate 结构能否等效替代数倍的算力增长"——如果 Afterstate 在低 Rollout 预算下的策略熵就低于 Standard 在高预算下的熵，说明解耦结构本身就是一种算力放大器。

### 5.3 TD 学习

在前两组实验中，评估器是固定的（Heuristic 由人工预设，N-Tuple 由离线预训练冻结）。TD 学习彻底移除了"给定评估器"这一前提——AI 从零开始，每走一步就更新一次自己的权重，在 25,000 局（Phase 1 基准）至 100,000 局（Phase 1 扩展、Phase 3）对弈中自学成才。

**核心目的：** 在无模型强化学习（从零训练）中，彻底厘清 Afterstate 的优势到底是因为"更新目标纯净（Bellman 公式好）"，还是因为"特征表征平滑（N-Tuple 输入好）"。

#### 5.3.1 算法与 Afterstate 实现

Phase 1 包含 5 组消融实验（3-A 至 3-E），围绕两个维度进行消融：更新目标（Q(s,a) vs V(s')）和 N-Tuple 特征输入（看 S vs 看 S'）。Phase 3 在此基础上去掉 Q-learning 基线（3-A, 3-B），新增 TDA-Full 精确期望实验（3-F）和 Downside-MV 双头方差实验（3-G），并将训练局数扩展至 100K。两个消融维度恰好对应 Afterstate 的两个理论优势——Bellman 公式更纯净对应更新目标的选择，输入盘面不含随机噪声对应特征输入的选择。

| 实验 | 更新目标 | 特征输入   | 类别                                          |
| ---- | -------- | ---------- | --------------------------------------------- |
| 3-A  | Q(s,a)   | State      | 传统基线——教科书级 Q-Learning                 |
| 3-B  | Q(s,a)   | Afterstate | 表征解耦——只换输入，目标不变                  |
| 3-C  | V(s')    | State      | 逻辑证伪——V(S) 与动作无关，退化为只比即时奖励 |
| 3-D  | V(s')    | Afterstate | **完全解耦——输入和目标全部解耦**              |
| 3-E  | MV(s')   | Afterstate | 风险控制——在 V(s') 基础上增加方差惩罚项       |

#### 5.3.3 专属评价指标

**TD 误差波动度 (TD Error Volatility)**：

训练过程中记录每步 TD 误差 $\delta = |R + \gamma V(s'_{\text{next}}) - V(s')|$，每 1000 步计算均方根：

$$\text{RMS}(\delta) = \sqrt{\frac{1}{n}\sum_{i=1}^{n} \delta_i^2}$$

此外，为消除不同实验间绝对分数量级差异的影响，同步计算**归一化 TD 误差 RMS**：

$$\text{Normalized RMS}(\delta) = \frac{\text{RMS}(\delta)}{\text{average\_score} + 1}$$

归一化后，3-A（得分 ~2500）和 3-D（得分 ~5200）的 TD 误差可直接比较，避免高分段 TD 误差的假性膨胀。

预期：3-A（State 输入）误差波动极大——环境随机落子的噪声直接混入输入，导致权重更新方向来回震荡，难以收敛。而 Afterstate 组（3-B, 3-D）的误差波动应迅速下降并趋于平稳。该指标是收敛速度的晴雨表：误差波动曲线的平滑度直接从网络更新机制上解释"为什么 Afterstate 收敛速度大幅领先"。

**经验过估计偏差 (Empirical Overestimation Bias)**：

经典 Q-Learning 的更新目标含 max 算子——从含噪声的估值中挑最大的。噪声天然有正有负，max 总是挑到噪声往上偏的那个，产生系统性高估（"自我感觉良好"）。Afterstate 将 max 从含噪声的 Q(S_next, a') 移到确定性的 R_a' + V(S'\_a') 上，理论上能压制过估计。

训练到第 90,000 局（权重基本稳定）时，随机截取 100 个盘面，记录网络预测价值 $V_{\text{predict}}$，随后让 AI 按当前策略实际玩完，计算真实折扣总得分 $G_{\text{real}}$：

$$\text{Bias} = V_{\text{predict}} - G_{\text{real}}$$

- 正 Bias → 盲目乐观（以为自己能赢，实际不行）
- 负 Bias → 保守安全

**偏差归一化的两种方式：**

传统的 Mean-of-Ratios（MoR，均值-of-比值）对每个盘面等权计算 $\text{mean}_i\left(\frac{\text{bias}_i}{\text{realized}_i + 1}\right)$。当个别盘面的 `realized` 极小时（如终局前仅得 0~2 分），分母接近 1，比值容易爆炸，一个离群点即可主导均值。

为此，new3 同步记录 **Ratio-of-Means（RoM，比值-of-均值）**：$\frac{\sum\text{bias}_i}{\sum(\text{realized}_i + 1)}$。RoM 对大返回值盘面天然赋予更高权重，对小盘面的噪声不敏感，是更鲁棒的归一化方法。两种归一化并行输出，供后续分析交叉验证。

**3-E 方差头误差。** 对于 MV(s') 实验，除 V-head 的 TD 误差外，new3 单独追踪 M-head（方差头）的 TD 误差 RMS（`td_error_m_rms`），以评估方差函数逼近的收敛质量——方差头学不好，惩罚项 $\lambda\sqrt{\text{Var}}$ 就缺乏意义。

该指标在 2048 的稀疏奖励条件下尤为重要——旧版（纯贪心+大 α）3-D 出现过估计爆炸（+193,795），揭示了 V(s') + Afterstate 在特定训练协议下的过估计风险。新版（ε+α 退火）在 25K 尺度压制了过估计，但 100K 扩展实验（§6.3.1）表明确过估计在长期训练中结构性回归（norm_bias_rom 从 −0.194 翻正至 +1.976）——过估计是否出现高度依赖训练协议与训练时长，这一发现直接影响了中层梯度谱分析（方向二）的动机强度。

### 6.1 Expectimax 结果

> 得分、达标率、耗时来自 100 局完整游戏；b_eff、压缩率来自搜索过程中每步实时统计。所有实验搜索深度统一为 2（1-A 为 1）。

| 实验 | 平均得分 | 2048达标率 | 4096达标率 | 步均耗时 | b_eff | 压缩率 |
| --- | --- | --- | --- | --- | --- | --- |
| 1-A (Greedy) | 4,779 | 0% | 0% | 0.11ms | 3.49 | 0.997 |
| 1-B (Standard+Heur) | 11,059 | 1% | 0% | 4.93ms | 47.23 | 0.996 |
| 1-C (After+Heur) | 11,952 | 4% | 0% | **0.64ms** | 14.45 | **0.504** |
| 1-D (Standard+StateNT) | 64,740 | 92% | 72% | 26.81ms | 39.59 | 0.998 |
| 1-E (Standard+AfterNT) | **5,066** | 0% | 0% | 34.47ms | 44.55 | 0.994 |
| 1-F (After+StateNT) | 60,126 | 93% | 67% | 2.56ms | 13.22 | 0.553 |
| 1-G (After+AfterNT) | **114,885** | **97%** | **91%** | **2.59ms** | 13.04 | 0.553 |
| 1-H (After+AfterNT+Pruning) | **125,721** | **100%** | **96%** | 2.31ms | **8.04** | 0.597 |

**关键发现：**

**1. 1-E 的灾难性崩塌——错配的代价。** 1-E（Standard 树 + Afterstate N-Tuple，得分 5,066）是本组最有价值的发现。Standard 树的 `leaf_mode="state"` 会在叶子节点模拟环境随机落子，将 S_next（含噪声的完整盘面）喂给打分器。但 Afterstate N-Tuple 的训练分布是纯净的 S'（落子前的中间盘面），从未见过这种含噪声的输入。输入分布的错位导致估值完全失真，得分从完全解耦的 1-G（114,885）暴跌了 **95.6%**，甚至不如纯贪心基线 Greedy（4,779）。这从反面铁证：**Afterstate 必须是一套完整的联合解耦方案——打分器训练分布和叶子评估对象必须一致，否则适得其反。**

**2. 错配成本的严重不对称。** 同为错配，1-F（Afterstate 树 + State N-Tuple，得分 60,126）仅比匹配的 1-D（64,740）下降约 7%。这种不对称性揭示了两种 N-Tuple 本质差异：State N-Tuple 训练时看的是已落子的完整盘面 S，当 `leaf_mode="afterstate"` 喂给它 S'（少了一个方块），它仍能从"几乎完整的盘面"中提取有效信息，适应性较强；Afterstate N-Tuple 训练时看的是纯净 S'，当 `leaf_mode="state"` 喂给它 S_next（多了一个随机方块），这个随机方块变成了它无法理解的噪声，估值系统彻底崩塌。State NT 从 S 泛化到 S' 容易（少一个方块不会改变盘面本质结构），Afterstate NT 从 S' 泛化到 S_next 困难（多一个随机方块污染了纯净信号）。

**3. 正确匹配时打分器质量主导得分。** 当打分器与 leaf_mode 正确匹配时（1-D vs 1-G），Afterstate N-Tuple（114,885）比 State N-Tuple（64,740）高出 **77%**。1-D 和 1-G 的区别是双重的——树结构不同 + 打分器不同——但 1-F vs 1-D 的对比（同 State NT、树不同）表明树结构对得分影响很小（60,126 vs 64,740）。因此 1-D→1-G 的 77% 增幅主要来自打分器质量的提升：看 S' 学到的估值函数比看 S 学到的更精确。

**4. 树结构主导效率，压缩率完美量化 DAG 折叠。** Afterstate 树（1-C/1-F/1-G/1-H）压缩率均为 0.50~0.60，Standard 树（1-B/1-D/1-E）压缩率均接近 1.0。Afterstate 树约一半节点被去重合并，这直接解释了耗时差异：1-D（不分离，26.81ms）vs 1-G（分离，2.59ms）加速 10.3 倍。Afterstate 树和打分器评估 S'（不需要逐空格模拟落子）两个因素叠加，带来了超过 10 倍的效率提升。

**5. 剪枝（1-H）锦上添花。** 1-H 在 1-G 基础上加入前向剪枝（保留 top-2 动作），得分再提 9% 至 125,721，达到 100% 2048 和 96% 4096。剪枝同时进一步压缩了搜索树（b_eff 从 13.04 降至 8.04），耗时微降。

| 2-G (AS 1000) | 1000 | 60,497 | 100% | 70% | 21.0 | 0.200 | 220.6ms |
| 2-D (Std 2000) | 2000 | 58,614 | 100% | 60% | 21.1 | 0.125 | 464.2ms |
| 2-H (AS 2000) | 2000 | **65,934** | **100%** | **80%** | 20.4 | 0.146 | 443.6ms |

**关键发现：**

**1. 分数水平整体大幅抬升。** 相比早期版本（27k~68k），新版 MCTS 全局分数提升至 45k~66k，底部（<30k）完全消失。这受益于 MCTS 动作选择策略的优化（从纯随机选择改为贪心+探索），即使仍然是人工价值函数的 Rollout，策略改进本身带来了显著收益。

**2. Afterstate 在高算力下占优，但反超阈值前移了。** 新数据中 Afterstate 在 1000 rollout 首次反超 Standard（60.5k vs 48.6k），到 2000 rollout 时扩大领先（65.9k vs 58.6k）。Afterstate 不再需要"极高算力才能反超"——1000 rollout 即已确立优势。这与早期版本中"2000 rollout Standard 碾压 Afterstate"的结论完全不同。

**3. Standard MCTS 在 500→1000 rollout 出现非单调下降。** 2-B (Std 500) → 2-C (Std 1000) 得分不升反降（58k→48.6k），降幅 16%。这种非单调是新增现象，此前版本中未见。可能原因：500→1000 的中间算力下 Standard 的 Rollout 噪声与 PUCT 的探索-利用平衡进入了不稳定区间。Afterstate 则没有出现这一下降（2-F 54k → 2-G 60k），表现平滑。

**4. 根方差和策略熵随算力持续收敛。** 无论是 Standard 还是 Afterstate，增加 Rollout 次数均使根方差从 ~25 降至 ~20，策略熵从 ~0.34 降至 ~0.12~0.15。算力增加使估值更集中、决策更确定——但 Standard vs Afterstate 在方差指标上的差异仍然不大。这与理论的预期一致：Afterstate 的方差缩减效应需要高质量评估器才能完全释放，在人工价值的 Rollout 下差异有限。

**5. 与 Expectimax 的对照：评估精度决定上限。** MCTS 纯人工价值的最好成绩（2-H: 65,934）远低于 N-Tuple Expectimax 的最好成绩（1-G: 114,885, 1-H: 125,721），但远高于 Heuristic Expectimax（1-C: 11,952）。这说明在缺少预训练评估器时，MCTS 通过大量即时采样弥补了评估质量的不足；但评估器的精度始终是理论上的上限瓶颈。

**关键发现：**

**1. 25K 局下过估计被退火压制，但只是延缓而非根治。** 所有实验的 average_bias 均为负值（−776 到 −5,365），norm_bias_rom 也全部为负（−0.142 到 −0.534）。+193,795 的巨大过估计在 ε+α 双重退火下消失——但这只是 25K 尺度下的现象。ε 退火在训练早期强制探索打破了"只走一条路→自我强化"的正反馈循环，α 退火在末期减小步长、不再翻旧账。**但后续 100K 扩展实验表明，过估计会在长期训练中结构性回归（见下文）。**

**2. 3-D 与 3-E 领跑，3-B 次之。** 经过 25k 局退火训练，3-E（MV）以 20,607 分和 36% 2048 率在数值上最高，3-D（V + Afterstate）以 19,508 分和 34% 2048 率紧随其后——这与旧版 new2 中 3-B > 3-D 的排序完全反转。Afterstate 表征 + V-head 的完全解耦方案在退火协议下展现出了理论预期的优势：共享经验的 V(s') 比四个独立 Q_head 学习效率更高。但由于 3-E 训练时间翻倍（3,315s vs 1,705s）且 2048 率仅领先 2 个百分点，方差惩罚在当前协议下增量有限，3-D 是性价比最优的选择。

**3. 3-A 与 3-C 得分接近（均约 3,000）。** Q(s,a) + State 和 V(s') + State 仍然很差，3-C 逻辑证伪继续成立——V(S) 对所有动作输出相同估值，退化为只比即时奖励。State 输入的噪声使得无论目标函数是 Q 还是 V，都无法进行有效的长期规划。

**4. TD 误差的绝对量级与分数正相关，归一化后可比。** 3-D 的绝对 TD 误差 RMS（527）远大于 3-A（106），但这只是因为 3-D 的分数（19.5k）远高于 3-A（3.7k）。归一化后（norm_td_rms_final），五个实验均在 0.026~0.034 之间，相差不大。这一现象提示：归一化 TD 误差对消融实验的区分度有限——它更适合追踪同一实验内的收敛趋势，而非跨实验比较。

**5. 3-E 的 M-head 在学但代价高。** 3-E 的 `td_error_m_rms_final=173.6` 说明 M-head 的 TD 误差在缓慢下降，但其量级与 V-head 相当（173 vs 538），且训练时间翻倍（3,315s vs 1,705s）。M-head 的学习率仅为 V-head 的 1/10，在 α 退火末期（α=0.002）实际更新步长仅 0.0002，后 20% 训练（5k 局）期间 M-head 基本冻结。方差惩罚 $\lambda\sqrt{\text{Var}}$ 因此缺乏可靠基础，5.6% 的分数提升难以判定是 MV 机制有效还是统计波动。

### 6.4 跨实验综合讨论

将三组实验的核心发现放在一起，一个清晰的图景浮现出来：

**When Afterstate wins, and when it doesn't.**

| 条件 | Afterstate 优势 | 机制 | 证据 |
| --- | --- | --- | --- |
| 评估器精确且正确匹配（AfterNT + leaf_afterstate） | **极大** | 打分器看 S' 杜绝评估噪声 + 树结构压缩 | 1-G vs 1-D：得分 +77%，耗时 -90% |
| 评估器精确但错配（AfterNT + leaf_state） | **灾难性负作用** | 输入分布错位，估值系统崩塌 | 1-E vs 1-D：得分 -92%，不如纯贪心 |
| 评估器精确但反向错配（StateNT + leaf_afterstate） | 轻微劣势 | State NT 对 S' 有一定适应性 | 1-F vs 1-D：得分 -7% |
| 评估器粗糙（启发式函数） | **中等** | 树结构压缩起效，但启发式精度低限制上限 | 1-C vs 1-B：得分 +8%，耗时 -87% |
| 无评估器、纯 Rollout 低预算 | **微弱** | Rollout 噪声 >> 树结构方差改善，AS 未占优 | 2-A (48.5k) vs 2-E (45.4k)：Std 略优 |
| 无评估器、纯 Rollout 中高预算 | **中等** | AS 在 1000+ rollout 确立优势，但方差差距仍不大 | 2-G (60.5k) > 2-C (48.6k)；2-H (65.9k) > 2-D (58.6k) |
| Afterstate 完全匹配 + 剪枝 | **最大化** | 解耦 + 剪枝协同，树更瘦决策更准 | 1-H：125,721 分，100% 2048，96% 4096 |
| 在线学习 25K（ε+α 退火） | **显著但收敛有限** | 完全解耦（3-D/3-E）领先，过估计被压制 | 3-D：19,508 (34% 2048)；3-E：20,607 (36% 2048) |
| 在线学习 100K（ε+α 退火） | **过估计回归，得分未提升** | 退火延缓但未根治过估计，norm_bias_rom 从负值翻正至 +1.98 | 3-D：18,731 (34% 2048, bias +1.98)；3-E 优势消失 |
| 在线学习 100K：TDA-Full | **无优势，劣于纯采样** | 精确期望消除采样噪声但引入目标不一致；过估计更严重 | 3-F：17,767（§6.6.1）；3-G（MV）：17,937，均 < 3-D |
| 在线学习 100K：Downside-MV | **无优势** | 下行方差惩罚压低探索积极性，未超越简单 V+Afterstate | 3-G：17,937 < 3-D 18,731（§6.6.1） |
| 环境漂移 P(4)→0.9 | **Afterstate 鲁棒性弱于 State** | Afterstate NT 内置 P(4)=10% 先验，环境偏移后估值失真；State 全宽展开天然覆盖 | M6 113k→19k (−83%)，M3 64k→11k (−82%)（§6.5） |

**三条核心结论：**

1. **Afterstate 的两个维度必须联合解耦——单独分离其中一个可能导致灾难。** 1-E 的崩塌（114,885 → 5,066）和 1-F 的轻微下降（64,740 → 60,126）揭示了：打分器的训练分布必须与树结构的叶子评估对象一致。只换打分器不换树（1-E），打分器收到从未见过的含噪声输入，估值完全失真。只换树不换打分器（1-F），State NT 虽然也能看 S'，但训练分布不同导致轻微性能损失。**两维度的错配成本高度不对称：Afterstate NT 对 S_next 的容忍度为 0（-96%），State NT 对 S' 的容忍度尚可（-7%）。**

2. **正确匹配时，Afterstate 的优势全面释放——但评估精度决定了释放程度。** 当打分器和 leaf_mode 都匹配时——1-G vs 1-D——Afterstate 方案带来 77% 的得分提升和 10 倍的效率提升。MCTS 中 Afterstate 在 1000 rollout 后确立优势，但增益幅度远小于 Expectimax（2-H vs 2-D 仅 +12%）。TD 学习中 3-D（完全解耦）和 3-E（MV）在退火协议下分别达到 19,508 和 20,607 分，完全解耦方案的 Afterstate 优势已确立。结论：Afterstate 的优势大小与评估信号的精度成正比——预训练 N-Tuple > 退火在线学习 > 即时 Rollout > 手工启发式。

3. **ε+α 退火延缓但未根治过估计；25K 局远未收敛。** 3-D 的 +193,795 过估计在 ε+α 双重退火下消失（25K 时 bias = −3,724），证明了旧版过估计是训练协议问题而非架构问题。但 100K 扩展实验揭示，过估计在长期训练中**结构性回归**——norm_bias_rom 从 −0.194 飙升至 +1.976，估值比实际回报高约 2 倍。退火只是压制了 max 算子偏差的累积速度，未能消除 V(s') 结构自身的过估计倾向。25K 局 + 退火是有限预算下的折中方案：收敛辅助而非收敛完成，在更长时间尺度上反而暴露了结构性缺陷。

### 6.5 Phase 2：环境漂移鲁棒性测试

Phase 2 的核心命题源于 Phase 1 的两项发现：Afterstate NT 对输入噪声的极度脆弱（1-E 崩塌 −92%），以及 Afterstate 估值的稳定性严重依赖评估精度。如果环境本身的概率分布发生了偏移——比如 P(4) 从标准的 10% 变成 90%——Afterstate 的解耦优势还能保持吗？Afterstate NT 将环境的先验概率「焊死」在了 N-Tuple 权重里，这是否意味着它在非平稳环境中的鲁棒性反而不如 State（State 每次决策都全宽展开所有可能的落子，天然地计算当前环境下的真实期望）？

Phase 2 在 Expectimax、MCTS、Q-Learning 三个算法框架下，人为注入 P(4) 从 0.1 渐变至 0.9 的环境漂移，检验两种状态建模方式的鲁棒性差异。

> **数据来源：** Expectimax: [final_results/phrase_2/expectimax/results/expectimax_drift_full_20260629_215319.csv](final_results/phrase_2/expectimax/results/expectimax_drift_full_20260629_215319.csv)；MCTS: [final_results/phrase_2/mcts_drift/results/mcts_drift_robustness_full_20260630_002832.csv](final_results/phrase_2/mcts_drift/results/mcts_drift_robustness_full_20260630_002832.csv)；Q-Learning: 数据为 smoke 模式（定性验证），数值精确但统计效力有限。

#### Expectimax 环境漂移

**完整漂移曲线（N-Tuple 组，50 局/概率点）：**

| P(4) | M3-State-NT | M6-After-ANT | M6 优势(分) | Disagreement Rate | Max Regret |
|---|---|---|---|---|---|
| 0.1 | 64,407 | **113,552** | +49,145 (+76%) | 0.26 | 15,464 |
| 0.3 | 54,239 | **69,746** | +15,507 (+29%) | 0.31 | 10,884 |
| 0.5 | 25,130 | **39,639** | +14,509 (+58%) | 0.31 | 18,679 |
| 0.7 | 18,767 | **26,216** | +7,449 (+40%) | 0.29 | 8,940 |
| 0.9 | 11,459 | **19,357** | +7,898 (+69%) | 0.33 | 13,839 |

**关键发现：**

**1. 初始猜想被推翻——Afterstate 的鲁棒性弱于 State。** Phase 1 的直觉是 Afterstate「解耦了环境」，应该在环境变化时更稳健。但实际数据恰恰相反：M6（Afterstate NT）的绝对优势从 P(4)=0.1 时的 +49,145 缩水至 P(4)=0.9 时的 +7,898。M6 得分下降 83%（114k→19k），M3 下降 82%（64k→11k）——衰减比例接近，但 M6 的绝对落差更大。Afterstate NT 并未展现出预期的「解耦 = 免疫环境」特性。

**2. 内在机理：Afterstate NT 将 P(4)=10% 的假设「过拟合」进了权重。** Afterstate 的 N-Tuple 训练时只见过 P(4)=10% 的环境，学到的 V(S') 估值隐式地假设了「空格的 90% 概率出 2」。当环境偏移后，这套估值逻辑完全失真。而 State 每次决策全宽展开所有可能的落子情况——它不依赖 P(4) 的先验，而是「算出」当前环境下的真实期望。在环境偏移时，这个「笨重但诚实」的机制反而提供了更强的鲁棒性。

**3. 决策分歧率随环境恶化单调上升。** Disagreement Rate 从 0.1 时的 0.26 上升到 0.9 时的 0.33——环境越恶劣，State 和 Afterstate 给出的动作建议分歧越大。M-Regret 在 P(4)=0.4 时达到峰值 19,819，说明在中等偏移下 Afterstate 的决策错判代价最高。

**4. 人工价值函数组（M1/M2）无区分度。** Heuristic State 和 Heuristic After 在任何 P(4) 下得分差距极小，两者都随环境恶化同步缓慢下降。低精度估值无法反映 N-Tuple 的过拟合效应。

| 0.9 | 13,977 | **15,410** | +10% | 0.367 | 0.363 |

**关键发现：**

**1. 两者同步衰减，Afterstate 在高偏移时优势不再稳定。** P(4)=0.7 时 Afterstate 甚至被 State 反超（14,871 < 15,797）。MCTS 的随机采样给了 State 一定的抗偏移能力——State 在搜索树中自然展开了更多随机分支，相当于为高 P(4) 环境做了「被动适配」。

**2. 微观树指标显示访问混乱度居高不下。** micro_entropy 在所有 P(4) 下维持在 0.36~0.40，没有明显收敛趋势。在极高随机性干扰下，MCTS 的 UCB 算力分配机制开始「雨露均沾」，无法锁定绝对优势分支。

**3. 人工价值函数的上限锁死。** 本组实验全程使用 FastHeuristic（手工价值函数），从未接入 N-Tuple。在 P(4)=0.1 时最优得分仅 35K——远低于 Phase 1 N-Tuple MCTS 的 136K（§6.6.3）和 N-Tuple Expectimax 的 115K。

#### Q-Learning 零样本环境漂移

> 25K 局训练收敛后的模型直接放入偏移环境，不做任何微调（Zero-Shot）。

| P(4) | 3-D (V+After) | 3-E (MV+After) |
|---|---|---|
| 0.1 | 19,767 | 17,957 |
| 0.3 | 10,215 | 10,733 |
| 0.5 | 7,809 | 7,556 |
| 0.7 | 4,644 | 5,991 |
| 0.9 | 5,242 | 5,141 |

**关键发现：**

**1. 全面崩溃。** 所有模型在 P(4)=0.9 时得分低于 6,000——不如 Phase 1 中最差的 3-A（3,672 分在标准环境下）。TD 目标在单步采样下，被恶劣环境的方差彻底撕裂，策略退化为极度保守或彻底混乱。

**2. 3-E（MV）没有展现出额外的鲁棒性。** 中层.md 预期 MV 的方差惩罚能使模型在恶劣环境下更稳健。但实际 3-E 在绝大多数 P(4) 下都落后于 3-D——方差惩罚反而在环境恶化时压低了探索积极性，使策略进一步保守化。

**3. 迁移学习（Continual Learning）未获得有效数据。** 文章思路.md 标注为「耗时极长、不建议继续完成」，final_results 中仅有 smoke 级别的定性验证。此项结论（「负迁移现象」——在困难环境中继续训练无法恢复原水平）是观察性描述而非量化结论。

### 6.6 Phase 3：架构演进与深度优化

Phase 2 暴露了 Afterstate 的两个核心短板：(1) N-Tuple 估值将环境概率过拟合进权重，导致环境漂移时鲁棒性崩塌；(2) 单步采样的 TD Target 在环境高方差下被撕裂。Phase 3 从三个方向发起针对性改进：Q-Learning 层面用精确数学期望替代单步采样（TDA-Full）并重构方差惩罚（Downside-MV）；Expectimax 层面利用 Afterstate 的 DAG 特性实现 HashDAG 去重和 Beam Search 剪枝；MCTS 层面接入 N-Tuple 估值并追踪树拓扑的微观变化。

#### 6.6.1 Q-Learning：TDA-Full 与 Downside-MV

> 全部实验均使用 100,000 局训练（ε 截断退火 0.10→0.0001，α 线性退火 0.05→0.002，seed=181），模型从零初始化。数据来源：[final_results/phrase_3/eval_results/qlearning_parallel_full_20260701_144225.csv](final_results/phrase_3/eval_results/qlearning_parallel_full_20260701_144225.csv)

**实验矩阵与结果：**

| 实验 | 目标模式 | 特征模式 | 得分 | 2048率 | 4096率 | TD RMS(fin) | Norm Bias(RoM) | 训练时间 |
|---|---|---|---|---|---|---|---|---|
| 3-C | V+采样 | State | 2,980 | 0% | 0% | 88 | −0.241 | 4,032s |
| 3-D | V+采样 | Afterstate | **18,731** | **34%** | 2% | 1,651 | **+1.976** | 13,115s |
| 3-E | TDA-Full | State | 2,980 | 0% | 0% | 158 | +0.502 | 12,434s |
| 3-F | TDA-Full | Afterstate | 17,767 | 20% | 2% | 1,314 | **+2.355** | 38,403s |
| 3-G | Downside-MV | Afterstate | 17,937 | 26% | 2% | 1,693 | +1.950 | 24,196s |

**关键发现：**

**1. TDA-Full 没有实现预期的「降维打击」——这是 Phase 3 最核心的反直觉发现。** 中层.md 预测 TDA-Full 会「从根源上消除环境白噪声」「TD-Error 收敛曲线变得极其平滑」。实际结果：3-F（TDA-Full+Afterstate）得分 17,767，**低于**采样版 3-D 的 18,731。3-F 的 2048 率 20% < 3-D 的 34%。3-F 的过估计 **更严重**（norm_bias_rom +2.355 > +1.976）。TDA-Full 的训练时间是 3-D 的 2.9 倍（38,403s vs 13,115s）。

**2. 失败原因分析。** TDA-Full 每步对 afterstate 做精确期望——遍历所有空格 × 两种 tile 概率（最多 28 次 board 操作）——这在数学上是「正确的 TD Target」。但问题出在两个环节：(a) TD(λ) 资格迹将当前步的精确期望广播到历史数百步的特征，而历史那些步的更新目标仍是采样版——目标函数的不一致破坏了收敛方向；(b) 精确期望消除了采样噪声，也使模型丧失了探索多样性，策略过早锁定次优解。唯一正面信号是 TD 误差 RMS 下降了 20%（1,314 vs 1,651）——精确期望确实让局部更新信号更稳定，但稳定在了「精确但错误的方向」上。

**3. Downside-MV (3-G) 也没有超过简单的 V+Afterstate (3-D)。** 3-G 得分 17,937 < 3-D 的 18,731，2048 率 26% < 34%。仅惩罚下行风险的方差设计（区分 up-variance 和 down-variance）并未带来策略质量的提升——下行惩罚反而压低了探索积极性，使模型过于保守。

**4. 3-E（TDA-Full+State）与 3-C（采样+State）得分完全相同（2,980）。** 这再次确认了 State 特征下 V 学习的根本缺陷：V(S) 与动作无关，退化为只比即时奖励，换什么目标函数都没用。

**5. 所有 Phase 3 QL 实验仍然没有 4096 突破。** 最高 4096 率 2%（3-D, 3-F, 3-G），100K 局训练仍然无法接近 Expectimax 冻结 N-Tuple 权重 91% 4096 的水平。

#### 6.6.2 Expectimax：Beam Search 与 HashDAG 优化

> 利用 Afterstate 的确定性特性，实现 HashDAG 缓存去重（无损）和 Top-2 Beam Search 剪枝（有损）。分别测试人工价值函数（Heuristic）组和预训练 N-Tuple 价值函数组。数据来源：depth=2 [search_optimizations_full_20260629_123634.csv](final_results/phrase_3/expectimax_eval_results/depth2/search_optimizations_full_20260629_123634.csv)、depth=3 [search_optimizations_full_20260629_150446.csv](final_results/phrase_3/expectimax_eval_results/depth3/search_optimizations_full_20260629_150446.csv)

| 配置 | Depth=2 得分 | Depth=2 2048率 | Depth=3 得分 | Depth=3 2048率 | Depth=3 4096率 |
|---|---|---|---|---|---|
| Heur + Base (全展开) | 12,505 | 6% | 20,096 | 36% | 0% |
| Heur + BeamSearch | 8,996 (−28%) | 0% | 13,344 (−34%) | 12% | 0% |
| Heur + HashDAG | 12,505 (≡Base) | 6% | 20,096 (≡Base) | 36% | 0% |
| NTuple + Base (全展开) | 113,399 | 98% | 144,703 | 100% | 84% |
| NTuple + BeamSearch | **120,719 (+6.5%)** | **100%** | 130,123 (−10%) | 100% | 76% |
| NTuple + HashDAG | 113,399 (≡Base) | 98% | 144,703 (≡Base) | 100% | 84% |

**关键发现：**

**1. HashDAG 在两种价值函数下均完美无损。** 所有配置下得分与 Base 完全一致（12,505≡12,505; 113,399≡113,399; 20,096≡20,096; 144,703≡144,703）。压缩率约 0.50~0.55（一半节点被去重合并），深度 3 时节点展开数减少 70%、时间缩短 53%。Afterstate 的确定性是 DAG 折叠的物理基础——不同动作路径高频收敛到同一个 S' 节点，哈希表命中率极高。

**2. Beam Search 在 N-Tuple depth=2 时反而提升了分数。** 中层.md 原文写「约 10% 的分数折损」——需要区分深度：depth=2 时 BeamSearch 得分 **+6.5%**（113,399→120,719），2048 率从 98% 升至 100%；depth=3 时才是 −10%（144,703→130,123）。浅层搜索时 Top-2 剪枝不仅没有错失关键分支，反而排除了估值噪声导致的分心路径，起到了「去噪」的意外正面效果。深层搜索时这个红利消失，剪枝代价开始显现。

**3. Heuristic 下 Beam Search 灾难性下降。** 人工函数在 depth=2 时 −28%，depth=3 时 −34%，验证了 Phase 1 的判断：「人工函数的盲目自信会剪掉生路」。高精度估值是剪枝的灵魂——N-Tuple 赋予了 Afterstate 将 Beam Search 从「豪赌」变为「自信」的判断力。

#### 6.6.3 MCTS：N-Tuple 接入与树拓扑分析

> 将 Phase 1 MCTS 中的人工价值函数替换为预训练 N-Tuple，并在高压残局（空格 ≤ 3）上进行微观拓扑追踪，剖析 Afterstate 和 State 的搜索树形态差异。数据来源：[mcts_topology_analysis_full_20260630_183925.csv](final_results/phrase_3/topology_tests/mcts_topology_analysis_full_20260630_183925.csv)

| 配置 | 得分 | 2048率 | 4096率 | macro_depth | probe_entropy | probe_depth |
|---|---|---|---|---|---|---|
| Heuristic + State | 29,276 | 68% | 6% | 5.22 | 0.221 | 5.37 |
| Heuristic + Afterstate | 33,272 | 82% | 10% | 5.83 | 0.235 | 6.05 |
| N-Tuple + State | 90,636 | 100% | 24% | 5.20 | **0.330** | 5.69 |
| N-Tuple + Afterstate | **135,712** | **100%** | **70%** | **5.71** | 0.318 | **6.29** |

**关键发现：**

**1. N-Tuple 接入后分数跃升 3~4 倍。** Heuristic State 29K → N-Tuple State 91K（×3.1），Heuristic After 33K → N-Tuple After 136K（×4.1）。Phase 2 末的推测被证实：「MCTS 的上限被人工价值函数锁死，需换 N-Tuple 才能释放 Afterstate 的拓扑优势」。N-Tuple Afterstate 的 136K 甚至超越了 Phase 1 N-Tuple Expectimax 完全解耦的 115K——MCTS 在拥有高精度估值后，其动态采样机制反而比固定深度的穷举搜索更有优势。

**2. 「State 矮胖，Afterstate 瘦深」的拓扑差异被精确量化。**
- State N-Tuple：probe_entropy=**0.330**（高混乱，算力分散在多条分支），macro_depth=5.20（较浅）
- Afterstate N-Tuple：probe_entropy=**0.318**（低混乱，算力聚焦），macro_depth=5.71（更深）
- Afterstate 的 layer_profile 在深层（第 5 层以后）节点数远少于 State，树形更「瘦」——UCB 算法在 Afterstate 的低方差估值下迅速收敛，将 90%+ 的算力绝对聚焦于正确的求生分支，向下挖掘更深。

**3. Heuristic 组的拓扑差异微弱。** State 和 Afterstate 在 Heuristic 下的 entropy 差异仅 0.014（0.221 vs 0.235），远小于 N-Tuple 组的 0.012。低精度估值无法发挥 Afterstate 的降噪优势——树结构的差异被估值的噪声淹没了。

**4. Afterstate 在 MCTS 中的终极意义被揭示。** Afterstate 通过降低估值方差，欺骗了 UCB 公式中的探索项，强行将 MCTS 从「广度优先采样」扭转成了「深度优先规划」。这就是它能在拥挤盘面下规避低级随机死亡的根本原因。

### 7.1 浅层发现（Phase 1 + 100K 扩展）

Phase 1 的三组消融实验在标准 2048 环境下对 Afterstate 的理论声称进行了逐维验证。结果既确认了理论的基本方向，也暴露了理论未能预见的问题。100K 局扩展实验进一步修正了部分 25K 时期结论。

**1. 两个维度可分离，但必须联合解耦——单维度分离可能适得其反。** Expectimax 的 2×2 矩阵揭示了 Afterstate 两个维度的复杂交互：树结构是否分离（`use_afterstate`）和叶子评估对象是否纯净（`leaf_mode`）不仅概念上可分，且必须与打分器的训练分布保持一致。1-E 证明了如果只换打分器不换树结构——Standard 树的 `leaf_mode="state"` 将 S_next 喂给训练时只看过 S' 的 Afterstate N-Tuple——得分从 1-D 的 64,740 崩塌到 5,066（-92%），甚至不如纯贪心基线。反方向的错配（1-F：Afterstate 树 + State N-Tuple）仅有 7% 的性能下降，揭示了错配成本的高度不对称：Afterstate NT 对输入噪声异常脆弱，State NT 则具有一定跨分布泛化能力。

**2. Afterstate 的增益随评估精度单调递增，但并非评估精度低时完全消失。** 从预训练 N-Tuple → N-Tuple MCTS → 退火在线学习 25K → 即时 MCTS Rollout → 手工启发式，评估精度逐级下降，Afterstate 的得分增益从 +77%（1-D→1-G）→ +49%（N-Tuple MCTS State→After）→ +85%（3-B→3-D）→ +12%（2-D→2-H）→ +8%（1-B→1-C）。

**3. ε+α 退火延缓但未根治过估计；100K 局时过估计结构性回归。** 25K 时 norm_bias_rom 为 −0.194（低估），100K 时飙升至 +1.976（高估约 2 倍）。退火只是压制了 max 算子偏差的累积速度——V(s') 结构的过估计倾向是结构性的，需要在 Bellman 目标层面（而非训练协议层面）解决。

**4. 100K 局训练未带来得分提升、未接近收敛。** 3-D 从 19,508 降至 18,731；3-E 从 20,607 降至 18,768，3-E 在 25K 时的微弱优势完全消失。唯一的进步是 4096 率破零（2%），但相比 Expectimax 同等 N-Tuple 权重的 91% 4096 差距悬殊。100K 远非收敛终点。

### 7.2 中层发现（Phase 2 + Phase 3 实际结果）

Phase 1 实验结束后导出了两个中层研究方向：环境漂移鲁棒性测试（方向一）和网络梯度谱分析（方向二）。Phase 2 完成了方向一的全部实验，Phase 3 补充了 Q-Learning 架构演进（TDA-Full / Downside-MV）、Expectimax 优化（BeamSearch / HashDAG）和 MCTS N-Tuple 拓扑分析。方向二（梯度谱分析）尚未执行。

**中层方向一（环境漂移）的核心结论：**

**1. Afterstate 的鲁棒性弱于 State，初始猜想被推翻。** Phase 1 提出「Afterstate 解耦环境 = 免疫环境变化」的假设。Phase 2 三种算法的一致结论是：Afterstate NT 将标准环境 P(4)=10% 的先验概率「过拟合」进了 N-Tuple 权重，环境偏移后估值系统失真。State 的全宽展开机制反而提供了更强的抗偏移能力。Expectimax：M6 从 114K 降至 19K（−83%），M3 从 64K 降至 11K（−82%），衰减比例几乎一致。MCTS：Afterstate 在 P(4)=0.7 时被 State 反超。Q-Learning：所有模型在 P(4)=0.9 时全面崩溃至 <6K。

**2. 环境漂移测试揭示的核心矛盾：「解耦环境 ≠ 免疫环境」。** Afterstate 通过状态截断确实在计算效率上取得了巨大成功（Phase 1: −90% 耗时），但在状态建模过程中，它将环境的概率分布吸纳进了自身的价值函数。环境静态且符合预期时 = 降维打击；环境动态或发生漂移时 = 作茧自缚。

**3. Q-Learning 零样本迁移完全失败。** 标准环境训练 25K 局的模型丢入偏移环境后，全部退化至随机策略级别（<6K）。MV 方差约束未提供任何额外的鲁棒性。

**中层方向二（Phase 3 Q-Learning 架构演进）的核心结论：**

**4. TDA-Full 精确期望没有实现预期的「降维打击」，反而劣于简单采样版。** 3-F（TDA-Full）在 100K 局下得分 17,767 < 3-D（采样版）的 18,731，过估计更严重（+2.36 vs +1.98）。精确期望消除了采样噪声但带来了两个新问题：TD(λ) 资格迹导致的目标函数不一致（精确期望 ← 采样历史），以及探索多样性的丧失。

**5. Downside-MV 也未超越简单 V+Afterstate。** 3-G（Downside-MV）得分 17,937 < 3-D 的 18,731。仅惩罚下行风险的方差设计在长期训练中没有累积优势——反而因过度保守压低了探索积极性。

**中层方向二（Phase 3 Expectimax 优化）的核心结论：**

**6. HashDAG 缓存去重完美无损，得分与 Base 完全一致。** Afterstate 的确定性使不同动作路径高频收敛到同一个 S'，压缩率约 0.50~0.55，深层搜索时压缩效果更强。

**7. Beam Search 效果高度依赖评估精度和搜索深度。** N-Tuple depth=2 时 BeamSearch 反升 +6.5%（「去噪」正面效果），depth=3 时 −10%。Heuristic 在任何深度都 −28%~−34%（「盲目自信剪掉生路」）。

**中层方向二（Phase 3 MCTS 拓扑）的核心结论：**

**8. N-Tuple 接入后 MCTS 分数跃升 3~4 倍，超越 Expectimax 完全解耦。** N-Tuple Afterstate MCTS 达到 136K（+70% 4096），超越了 N-Tuple Expectimax 的 115K——MCTS 的动态采样机制在获得高精度估值后比固定深度穷举搜索更有优势。

**9. 「State 矮胖，Afterstate 瘦深」的拓扑差异被精确量化。** State 的 probe_entropy 更高（0.330 vs 0.318）、macro_depth 更浅（5.20 vs 5.71）。Afterstate 通过降低估值方差，欺骗了 UCB 的探索项，将 MCTS 从广度优先采样扭转成深度优先规划——这是它规避低级随机死亡的根本原因。

### 7.3 通往深层（Belief-Afterstate POMDP）的桥梁

一切解耦优势的前提是"AI 能清晰看到 S 以计算 S'"。一旦加上战争迷雾（POMDP），所有第一阶段验证过的优势瞬间归零。这迫使引入 **Belief-Afterstate**——基于概率分布的"盲猜"新架构。

### 7.4 关键局限性

1. **Q-Learning 过估计未根治，100K 局也未收敛。** 100K 扩展实验证实：ε+α 退火仅延缓而非根治过估计（norm_bias_rom 从 −0.194 飙至 +1.976）；得分随训练量不升反降；4096 率最高仅 2%，距 Expectimax 冻结权重的 91% 差距悬殊。TDA-Full 和 Downside-MV 两类架构改进均未在 100K 尺度上超越简单的 V+Afterstate 采样版。更长的训练量（200K+）或去掉 TD(λ) 资格迹后的 TDA-Full 仍有待验证。

2. **MCTS Rollout 策略在 Phase 1/2 中为纯启发式**：Phase 3 的 MCTS N-Tuple 拓扑实验已将 Rollout 替换为预训练 N-Tuple 估值，但 Phase 1 和 Phase 2 的 MCTS 数据全部基于 FastHeuristic（手工价值函数）Rollout，限制了绝对得分的上限（Phase 1 最高 66K vs Phase 3 N-Tuple MCTS 136K）。

3. **Phase 2 QL 漂移为 smoke 数据**：Zero-Shot 测试仅跑了 smoke 级别的定性验证（少量对局），无法产出有统计显著性的量化结论。迁移学习（Continual Learning）实验未完成。

4. **TDA-Full 与 TD(λ) 资格迹的交互未充分探究**：Phase 3 代码中 TDA-Full 仍使用 TD(λ) 资格迹（λ=0.5），导致精确期望目标与采样历史之间不一致。若改为纯 TD(0)（λ=0.0）或单独为 TDA-Full 设计独立的资格迹衰减方案，结果可能有本质不同。

5. **搜索深度固定为 2（Phase 1）/ 3（Phase 3 Expectimax）**：更深的搜索可进一步放大 Afterstate 的 DAG 折叠优势，目前的深度限制可能低估了 Afterstate 在极限算力下的潜力。

6. **环境噪声固定（P(4)=10%），完全信息假设**：Phase 2 已测试概率偏移但不涉及重新训练；深层研究的 POMDP（战争迷雾）环境尚未启动。

## 附录二：3-E MV(s') 实验设计（已执行，结果见 §6.3）

3-E 为 TD 学习消融实验的风险控制组，已完成方案设计、代码实现并产出结果（25,000 局训练）。

**设计动机。** 3-D（V(s') + Afterstate）在理论上最为优美——统一的 V(s') 让所有动作共享经验，且评估对象不含随机噪声。但经典 Q-Learning 的 max 算子存在系统性过估计风险：从含噪声的估值中取最大值会天然偏向正噪声。Afterstate 虽然将 max 从 $Q(S_{\text{next}}, a')$ 移至 $R_{a'} + V(S'_{a'})$，但在 2048 的稀疏奖励条件下，V(s') 的 max 是否仍会过估计，是一个需要实证检验的问题。

**25K 局结果（§6.3）**：在 ε+α 双重退火下，旧版（纯贪心+大 α）的 +193,795 过估计在 25K 时消失（bias = −3,724），3-E 比 3-D 得分微增 5.6% 但训练时间翻倍。MV 在当前协议下贡献有限，M-head 在 α 退火末期基本冻结（实际 α≈0.0002）。

**100K 局扩展（§6.3.1）**：过估计结构性回归——norm_bias_rom 从 −0.194 翻正至 +1.976。3-E 的得分优势完全消失（18,768 vs 3-D 18,731，差 0.2%）。ε+α 退火延缓但未根治过估计，V(s') 结构的过估计倾向是结构性的。

**后续（Phase 3, §6.6.1）**：Phase 3 的 Downside-MV (3-G) 将 M 拆分为 m_up_head 和 m_down_head 分别追踪正向和负向方差，在 100K 局下得分 17,937，仍未超越简单 V+Afterstate 采样版 3-D 的 18,731。

**机制。** 在 V 之外增加一个 M，学习 $\mathbb{E}[G^2 \mid S']$（$G$ = 从当前状态往后直到终局的总得分），与 $V(S') \approx \mathbb{E}[G \mid S']$ 组合得到方差 $\text{Var}(S') = \max(0, M - V^2)$。动作选择时减去 $\lambda \cdot \sqrt{\text{Var}}$，惩罚高波动方向：

$$
\begin{aligned}
M(S') &= \mathbb{E}[G^2 \mid S'] \\
\text{Var}(S') &= \max(0,\, M(S') - V(S')^2) \\
a^* &= \arg\max_{a}\left[ R_a + V(S'_a) - \lambda \cdot \sqrt{\text{Var}(S'_a)} \right]
\end{aligned}
$$

M 的学习率设为 V 的 1/10——$G^2$ 比 $G$ 波动大得多，降低学习率防止 M 的训练不稳定干扰 V。

**3-E 独立指标。** new3 为 3-E 单独追踪 M-head 的 TD 误差 RMS（`td_error_m_rms`），与 V-head 的 TD 误差（`td_error_rms`）并列输出。方差函数逼近的质量是 MV 策略有效性的前提——如果 M-head 学不好，$\lambda\sqrt{\text{Var}}$ 惩罚项就沦为噪声注入。两套 TD 误差的收敛曲线对比（见 `variance_head_td_error_smoothed.png`）直接回答：方差头是否比价值头更难学？是否需要降低 M-head 学习率？
