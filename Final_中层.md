# 核心重构工作流：基于 Afterstate 边界裕度的安全前向过滤 (Margin-Based Forward Pruning)

## 第一阶段：理论修正与问题界定 (The Rationale)

### 1. 历史方案的局限：硬性 Top-K 截断的风险
在我们最初的设计中，采用了强行截断 Top-2 动作的策略。
* **逻辑缺陷**：这是一种“一刀切”的危险启发式。在 2048 的复杂中局里，往往有 3 个甚至 4 个动作的预期价值极为接近。强行只保留 2 个，极易砍掉能够化解危机的“唯一解”，导致搜索完备性被破坏，得分产生不可控的异常波动。
* **算力浪费**：在必胜或必败的绝对明朗局势下（例如只有一个动作能存活），Top-K 策略依然会傻傻地去深搜 2 个动作，浪费了宝贵的计算资源。

### 2. Alpha-Beta 的概率陷阱
如前所述，由于 Expectimax 中存在环境概率生成节点（Chance Node），传统的 Alpha-Beta 剪枝在面对“极小概率 $\times$ 极大价值”的长尾期望时，无法建立有效的收敛边界，导致深层子树基本无法被实质性剪除，时间复杂度没有得到根本改善。

### 3. 破局之道：安全边界过滤 (Margin-Based Pruning)
针对 N-Tuple 网络估值极其精准的特性，我们采用工业界成熟的**安全前向过滤**。
* **核心理念**：我们不再固定保留几个动作，而是建立一个**动态的宽容阈值（Margin）**。
* **逻辑支撑**：如果一个动作在浅层（Depth=0）的即时估值，比当前最佳动作的估值落后了一个“不可逾越的鸿沟”（例如相差几千上万分），那么无论后续随机环境如何配合，这个动作在深层搜索中也绝对无法翻盘。将其剪除是**统计学意义上的绝对安全**。

---

## 第二阶段：核心执行步骤与逻辑流 (The Execution)

本阶段完全剥离具体代码，专注于决策节点（Max Node）在发起深层搜索前的“安全过滤”调度逻辑。此逻辑主要作用于根节点或树的较浅层。

### 模块 A：浅层基线扫描 (Shallow Baseline Evaluation)
目标是利用最低的计算成本，摸清当前盘面下所有合法动作的价值底色。
1. **合法动作遍历**：获取当前盘面的所有合法动作（上、下、左、右）。
2. **生成即时快照**：对每个合法动作，生成对应的确定性 Afterstate 盘面，并记录该动作产生的即时奖励（Reward）。
3. **极速估值读取**：调用 N-Tuple 网络，对所有的 Afterstate 盘面进行 $O(1)$ 的静态估值。
4. **计算初始期望**：将每个动作的即时奖励与网络估值相加，得到一组浅层基准分数（Immediate Value）。

### 模块 B：动态阈值裁决 (Dynamic Thresholding & Pruning)
目标是确立“安全线”，剔除绝对劣解。
1. **锁定全局最优**：在模块 A 得到的所有浅层基准分数中，找出最高分，将其定义为 $V_{max}$。
2. **引入宽容裕度 (Margin)**：设定一个固定的经验常数 $Margin$（例如 5000 分或 10000 分，需根据 N-Tuple 的分数尺度进行超参数调优）。
3. **确立安全底线**：计算安全阈值 $Threshold = V_{max} - Margin$。
4. **执行绝对抹杀**：
    * 遍历所有合法动作。
    * 任何浅层基准分数**严格小于** $Threshold$ 的动作，被判定为“不可挽回的劣解”，直接从候选队列中永久剔除（剪枝）。
    * 剩余的动作，无论数量是 1 个、2 个还是 4 个，全部保留。

### 模块 C：按需深层探索 (Adaptive Deep Search)
目标是将宝贵的算力完全倾斜给真正具有竞争力的动作。
1. **自适应下潜**：对模块 B 中幸存下来的动作，依次发起完整的、不打折扣的深度 Expectimax 搜索（展开后续的 Chance Node 与 Max Node）。
2. **汇总归拔**：对比这些幸存动作经过深层搜索后返回的真实期望价值，选出最终的决策动作。

>其实对于上面的可以进行一定的修改,原本的方法类似于beam search,不过这应当是建立于我们具有极为稳健的价值评判函数才可以这样处理.我们的希望是将其从硬性的beam search,转变为具有一定缓冲的搜索.

>不过可能会导致两个问题:第一,时间耗时可能并不会有明显降低,反而有所提升,因为低深度下可能会导致判断的消耗更多

>如果如上的实现难以进行,有一个确保能够实现的降低时间的方法--储存盘面价值的哈希表,我们能发现afterstate的重合率远高于state,因此,通过哈希表以O(1)的方式提取后续的价值会快于计算的时间.

## 第三阶段：预期效果与验收指标 (The Validation)

>如果使用哈希表储存的方式，其实指标和之前是一样的。

实施安全边界过滤后，系统将展现出高度的“弹性”与“智能”，请通过以下指标进行验收调优：

1. **得分的稳健回升 (Score Stability)**
   * **现象**：相较于硬性 Top-2 截断，此方案不会误杀分数接近的高潜动作。最终游戏平均得分应稳定回升，持平或极其逼近完全不剪枝的基础版本（如 1-G）。
2. **时间消耗的自适应弹性 (Adaptive Time Complexity)**
   * **现象**：单步决策时间将呈现明显的动态分布。
   * **顺风局/明显局**：当盘面有一个绝对优解（其他动作会破坏阵型导致估值暴跌）时，算法会瞬间剪掉另外 3 个动作，仅对 1 个动作进行深搜，耗时极低。
   * **逆风局/复杂局**：当盘面极度复杂、多个动作分数咬紧时，算法会自动保留 3-4 个动作进行深搜以确保安全，此时耗时回归正常上限。
3. **裕度超参的敏捷度 (Margin Hyperparameter Tuning)**
   * **调优验收**：需要通过网格搜索确定最佳的 $Margin$ 值。
   * 如果 $Margin$ 设置过大，过滤形同虚设，时间降不下来；
   * 如果 $Margin$ 设置过小，退化为 Top-1 贪心算法，分数将出现断崖式下跌。
   * 完美的 $Margin$ 应能在“砍掉 50% 以上无效分支”与“保持 99% 的 SOTA 得分”之间找到甜点。

# 核心重构工作流：2048 强化学习从“单步采样”向“全宽度期望更新 (TDA-full)”的演进

## 第一阶段：问题界定与痛点剖析 (The Diagnosis)

### 1. 历史代码的逻辑陷阱：单步蒙特卡洛采样 (Single-Step Sampling)
在早期的 Q-Learning 训练代码中，我们计算目标值（TD Target）的逻辑是：
* 智能体在状态 $S_t$ 执行动作 $a_t$，盘面滑动合并。
* 调用 `popup_with_rng(next_b, rng)` **随机生成**一个 2 或 4。
* 获取新状态 $S_{t+1}$，计算 $Target = Reward + \gamma \max Q(S_{t+1}, a')$。

**致命缺陷：方差爆炸 (Variance Explosion)**
2048 的环境具有极高的随机性（例如 15 个空位 $\times$ 2 种数字 = 30 种可能性）。单次 `popup_with_rng` 只采样了这 30 种可能中的**一种**。如果运气好方块刷在角落，Target 就极高；如果刷在中间破坏阵型，Target 就暴跌。这种高方差导致神经网络接收到的梯度信号充满噪音。

### 2. 实验现象的印证：高估偏差 (Overestimation Bias)
我们在并行日志中观察到（如实验 3-D：`V(s')+AfterstateNTuple`），虽然平均得分飙升至 19508，但其 `average_bias` 却恶化到了惊人的 **-3723**。
* **原因分析**：强化学习中的 $\max$ 操作本身就会放大高估偏差。当底层数据因为“单步采样”充满了高方差的随机噪音时，$\max$ 操作会贪婪地捕获那些“因为运气极好而被高估”的异常值，最终导致整个网络对所有阵型的估值严重虚高，最终引发策略崩溃（这也是为什么文献中提到 TDA-all 容易训练失败的原因）。

---

## 第二阶段：理论破局与演进思考 (The Rationale)

### 1. 解决思路的诞生：环境模型的绝对掌控
强化学习中解决高方差的终极武器是**期望更新 (Expected Update)**。
普通的强化学习环境（如雅达利游戏）是一个黑盒，智能体不知道环境运作规律，只能靠采样。但 **2048 是一个白盒环境**（White-box Environment）：
* 我们完美知道生成规律：空位生成概率均等，生成 2 的概率为 90%，生成 4 的概率为 10%。
* **破局点**：我们完全可以废除 `popup_with_rng`，直接用代码遍历所有可能的生成结果，用严格的数学期望（加权平均）来取代一次性的掷骰子。

### 2. 核心争论：State 还是 Afterstate？
既然都要算期望，State 和 Afterstate 都能算，为什么我们要强烈向 Afterstate 倾斜？
* **形态学纯净度 (Morphological Purity)**：N-Tuple 网络提取的是盘面的单调性和平滑度。如果用 State 输入网络，盘面上会多出一个尚未被玩家处理的随机方块，这个方块相当于**图像上的噪点**，会严重干扰网络对“玩家刚做完决策后的阵型”的真实评估。Afterstate 是方块合并后、新方块生成前的状态，最为纯净。
* **零样本泛化 (Zero-Shot Generalization)**：在 2048 中，不同的 State 执行不同的动作，极有可能殊途同归到达同一个 Afterstate。如果我们对这个 Afterstate 进行了期望更新，就等于同时更新了所有能到达它的 State 的价值，数据利用率呈指数级跃升。

## 第三阶段：具体更新机制的严格界定 (The Mechanism)

在这个重构版本中，我们必须严格区分 $Q(s, a)$（状态模式）和 $V(s')$（后状态模式）在**什么局面下被调用**，以及**如何获取它们的 TD Target**。

### 1. 模式 A：状态的期望更新 (Expected Sarsa / 对应未来实验 3-F)
* **适用局面 (State)**：玩家正准备做决定时的盘面（方块已生成，等待玩家输入）。
* **训练实体**：$Q(s, a)$，即评估“在特定盘面下采取特定动作”的好坏。
* **获取流程 (TD Target 的计算)**：
  1. 在 $S_t$ 执行动作 $a_t$，得到临时滑动的盘面（无新方块）。
  2. 获取临时盘面所有的空位数量 $N$。
  3. 遍历每个空位，分别推演放置 2（权重 $0.9/N$）和放置 4（权重 $0.1/N$），得到一系列真实的次态 $S_{t+1}$。
  4. 对每个 $S_{t+1}$，遍历 4 个合法动作求最大值：$\max_{a'} Q(S_{t+1}, a')$。
  5. 将所有次态的最大值按权重累加，得到最终期望。

### 2. 模式 B：后状态的全宽度期望更新 (TDA-full / 对应未来实验 3-G)
* **适用局面 (Afterstate)**：玩家刚做完决定，盘面刚完成滑动合并，环境**尚未**生成新方块的瞬息快照。
* **训练实体**：$V(s')$，即评估“玩家打造出的这种结构骨架”的好坏。
* **获取流程 (TD Target 的计算 / 严格对照文献公式 3.2 与 3.3)**：
  1. 当前处于确定的 Afterstate $s'_t$。
  2. 获取所有空位数量 $N$。
  3. 遍历每个空位，分别放置 2（权重 $0.9/N$）和 4（权重 $0.1/N$），得到次态 $S_{t+1}$。
  4. 对每个次态 $S_{t+1}$，遍历其合法的玩家动作 $a_{t+1}$：
     * 执行该动作获得即时奖励 $R$ 和次级 Afterstate $s'_{t+1}$。
     * 读取网络估值：$V_{net}(s'_{t+1})$。
     * 找出该次态下最大的：$\max [R + V_{net}(s'_{t+1})]$。
  5. 将步骤 4 算出的最大值，按步骤 3 的概率权重累加求和，得到深度期望价值 $DV(s'_t)$。
  6. **更新执行**：让网络输出 $V(s'_t)$ 逼近 $DV(s'_t)$。

## 第四阶段：代码工作流的重构蓝图 (The Execution Flow)

为了在工程中实现上述转换，我们需要将原本的强化学习主循环和网络更新逻辑（即 `update_step` 函数）进行彻底替换。

### 步骤 1：废除随机采样逻辑
* 彻底删除核心计算路径中的 `popup_with_rng(next_b, rng)`。
* 将环境交互从“根据单步实际发生的情况更新”转变为“根据已知模型在内存中进行全景推演更新”。

### 步骤 2：重构 TD Target 计算核心模块
* 建立一个新的隔离函数 `_compute_expected_target(afterstate_raw)`。
* 内部逻辑：
  * 解析 `afterstate_raw` 的空位集合。
  * 如果没有空位，返回 0（或终局惩罚）。
  * 设概率基础权重 $W = 1.0 / len(空位)$。
  * `expected_value = 0.0`
  * 对每一个 `pos` in 空位：
    * 构造放置 2 的盘面：寻找最大子动作价值（奖励 + 次态估值），`expected_value += W * 0.9 * max_val`
    * 构造放置 4 的盘面：寻找最大子动作价值（奖励 + 次态估值），`expected_value += W * 0.1 * max_val`
* 返回精确无方差的 `expected_value`。

### 步骤 3：无缝接入主更新流
* 依然维持 N-Tuple 的 Replacing Traces (替换迹) 和特征映射逻辑不变。
* 在执行 `update_step` 时，将原来的：
  `target = reward + gamma * self.best_action_value(next_raw)`
* 替换为：
  `target = reward + gamma * self._compute_expected_target(after_raw)`

### 步骤 4：设计验证消融实验
为了通过严谨的数据证明重构的有效性，工作流需新增两组基准实验：
1. **实验 3-F (Expected Q + StateNTuple)**：证明期望计算本身能消除 `td_error_rms_final` 的抖动，并大幅缓解 `average_bias` 的爆炸。
2. **实验 3-G (TDA-full + AfterstateNTuple)**：证明在同等消除方差的条件下，Afterstate 凭借其在形态学上的纯净度，能在相同训练步数下达到比 3-F 高得多的 `average_score`，从而完成对 State 模式的决定性碾压。

### 指标：
#### 收敛稳定性指标 (Stability Metrics)：

1. TD Error RMS (均方根误差)：这是衡量网络预测是否收敛的最直接指标。重构后，TD Error RMS 应当比“单步采样”版本更小且震荡更轻微。

2. Normalized TD-Error：误差除以当前平均得分。因为游戏后期分数极高，绝对误差会随时间增长，归一化误差更能反映训练稳定性。

#### 偏差控制指标 (Bias Metrics)：

1. Empirical Overestimation Bias (高估偏差)：这是任务二最重要的指标（对照日志中的 average_bias）。TDA-full 引入后，此数值应显著向 0 趋近（趋近于 0 表示预测与实际达成的一致）。

2. Bias-to-Return Ratio (归一化偏差比)：通过 bias / realized_return 衡量，如果该比值随训练波动剧烈，说明存在严重的 Value Gap。

#### 策略泛化与利用率指标 (Generalization Metrics)：

1. Score-per-Episode (学习曲线斜率)：TDA-full 的优势在于通过期望更新获取了更纯净的梯度，预期学习曲线的爬升速度应显著快于采样版。

2. Afterstate 重合利用率：如果你的实现记录了 visited_state_count，观察重构后对于相同阵型的重复利用率（即学习一个阵型即掌握该阵型下的所有随机分支）是否提升。
