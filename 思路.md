# Afterstate 与 State 对比实验完整研究梳理
## 文档总览
本文围绕 **afterstate / state 两种状态建模方式**，分别在三类算法框架（N-tuple+Expectimax、MCTS、深度强化学习Q/V网络）中开展多阶段消融、环境偏移、算力对照实验，划分 Phase1（基准对照）、Phase2（环境鲁棒性测试）、Phase3（后续改进思路）三阶段，完整梳理实验现象、机理分析、现存缺陷与未来拓展方向。

---

## 一、核心基础概念：N-tuple 泛化设计思路
传统 N-tuple 仅能对固定棋盘数组做匹配，泛化能力弱、迭代学习轮次受限。
本文对 N-tuple 引入**数值线性泛化机制**：
> 例：模型学习到片段 `[2,4,8,16]` 的价值规律，可自动类比迁移至 `[4,8,16,32]` 等同结构、数值翻倍的棋盘片段，提升状态复用与泛化能力。

研究核心对比维度：
1. 状态建模：`afterstate`（玩家移动后、随机方块刷新前确定中间局面）vs `state`（包含随机方块完整原始棋盘）
2. 价值评估载体：人工手工价值函数 / 泛化N-tuple学习价值 / 深度网络Q/V价值
3. 三大算法测试平台：Expectimax、MCTS、深度强化学习

# 二、实验分模块完整梳理
## 模块1：Expectimax + N-tuple 价值函数实验（Phase1 / Phase2 / Phase3）
### 2.1 Phase1：基准消融对照实验（固定100轮迭代）
实验组划分：人工价值函数组、泛化N-tuple价值函数组
#### （1）人工价值函数实验结果
1. 得分表现：`afterstate`、`state` 最终平均得分差距极小，无法拉开性能差距；
2. 时间效率：同等条件下 `afterstate` 推理耗时显著低于 `state`；
3. 结论：仅依靠简陋人工估值，无法发挥afterstate价值区分优势，需要高精度学习型价值函数才能拉开分数差距。

#### （2）N-tuple 学习价值函数实验结果
1. 得分表现：符合预期，`afterstate` 平均得分显著优于 `state`，性能差距有效拉开；
2. 时间表现：`afterstate` 推理耗时大幅下降；
3. 时间效率提升内在机理
    - 计算层面：afterstate仅需评估单一确定中间局面，无需像state一样遍历全部随机分支求均值，减少大量重复估值计算；
    - 环境噪声层面：afterstate是纯动作确定局面，状态分布稳定，配套N-tuple估值噪声更低，预测置信度更高。

#### （3）衍生优化方案（基于afterstate稳定特性）
##### 方案1：Beam Search 分支剪枝替代完整遍历
- 设计逻辑：afterstate估值稳定性高，可近似认为评估结果可信度高；每一步仅保留Top2最优动作分支向下拓展，替代全分支搜索；
- 实验效果：
  - N-tuple估值场景：全局平均得分小幅提升，推理耗时显著降低，搜索深度越大加速效果越明显；
  - 人工估值场景：虽能缩短计算时间，但最终得分大幅下滑；
- 原因分析：人工函数评估偏差大，过度信任估值结果会导致算法错误舍弃关键分支，引发策略退化。

##### 方案2：哈希表缓存去重（DAG图优化）
- 设计逻辑：afterstate动作确定，多路径搜索会收敛至同一中间局面；使用哈希表存储局面-估值键值对，重复局面直接读取缓存，避免重复计算；
- 实现难度：低，可快速落地。

### 2.2 Phase2：环境偏移鲁棒性测试（先验方块生成概率与实际环境不一致）

原因：由于afterstate本身的平稳性，并且能够将环境和动作区别开的属性，我们认为其在环境困难或者环境有所误差的情况下具有更为良好的表现

#### （1）人工价值函数组

两类状态建模得分同步缓慢下滑，区分度极低，无有效参考价值，不作重点分析。

#### （2）N-tuple 学习价值函数组
1. 实验现象：环境概率偏移后，`afterstate` 得分衰减幅度远大于 `state`，鲁棒性更差；
2. 猜想机理：
   - afterstate 的N-tuple估值内置固定10%方块4生成概率假设，环境概率偏移后估值期望完全失真；因此想要进行分离
   - state原生需要遍历全部随机分支做均值，天然覆盖极端偏移场景；且本身估值噪声大，环境变化带来的得分衰减更平缓。

### 2.3 Phase3：后续改进思路（暂不落地实现）
1. 双N-tuple解耦设计：拆分**纯盘面价值评估模块** + **随机概率风险评估模块**，分开训练两套N-tuple；
2. 风险网络拟合：使用神经网络拟合与方块生成概率绑定的风险评估函数；
3. 未来目标效果：让afterstate在未知概率黑盒2048环境中，自适应调整期望估值，降低环境偏移带来的性能衰减。

## 模块2：MCTS 蒙特卡洛树搜索实验（Phase1 / Phase2）
全部实验默认使用**人工手工价值函数**，变量为算力规模、state/afterstate建模、环境偏移、动作选择策略。
### 2.1 Phase1：算力对照基准实验
1. 整体结论：MCTS基础得分远高于Expectimax纯搜索，算法本身具备极强优势；
2. 算力对照现象：
   - 低算力场景：afterstate、state得分接近，afterstate无优势；
   - 高算力场景：afterstate得分显著优于state；
3. 截断优化设计：使用人工价值做局面截断，解决2048无限对局、随机噪声过多问题；搭配随机rollout步数稳定估值；
4. 低算力无区分度原因：采样反馈样本不足，低概率危险局面（如10%概率生成4直接崩盘）无法充分采样，局面风险无法被完整识别；
5. 高算力优势来源：充分覆盖小概率随机事件，盘面价值收敛稳定；afterstate输出动作回报值更稳定，提升搜索置信度；
6. 时间特性说明：MCTS依靠大量随机rollout，afterstate在Expectimax中的计算加速优势无法体现。

### 2.2 Phase2：环境偏移 & 动作选择策略优化实验

>与之前基于同样的采样下进行。
#### （1）环境偏移测试
人工函数条件下，两类建模得分同步大幅下滑；算力越高，afterstate得分衰减幅度越大；
MCTS自身随机性强、实验轮次有限，该组数据无法得出有效确定性结论。

#### （2）MCTS动作选择策略优化
原始方案：完全随机分支拓展；
改进方案：融合贪心+探索策略，利用afterstate稳定估值分配算力分配权重，优先高价值分支拓展；
1. 实验现象：人工函数条件下，afterstate反而需要更高算力才能实现得分反超state；
2. 合理解释：afterstate的稳定优势高度依赖高精度价值函数，人工估值存在固有偏差，干扰搜索先验，需要更多采样弥补误差；

因此后续可以尝试使用Ntuple价值代替，预期在低算力就可以超越state很多


#### （3）未完成拓展方向（MCTS1）
1. 替换价值函数：将人工函数替换为泛化N-tuple估值，进一步提升上限；
2. 树结构特性猜想：afterstate估值置信度更高，生成的搜索树会更**瘦且深**，倾向深度长期规划；

#### （4）长期未落地思路
参考Expectimax双模块设计，拆分盘面收益、随机风险两套评估模型，构建鲁棒性更强的MCTS搜索框架（不准备实现）。

## 模块3：深度强化学习（Q/V价值网络）实验（Phase1 / Phase2）
### 3.1 Phase1：State / Afterstate 价值建模基础对比
1. 核心天然优势：
   - afterstate 可直接学习状态价值 $V(S')$；
   - 原始state无法高效拟合V值，效果极差；
   - Q学习需要存储大量状态-动作配对，参数量、存储开销远大于V网络；
2. 训练痛点：模型输出误差偏大，受限于训练对局数量，无法大规模迭代；
3. 改进方案：引入MV方差约束目标
   - 设计思路：仅对低于均值的负向误差计算损失，分配差异化损失系数；约束模型偏向保守稳健策略，优化长期预估；
4. MV方案缺陷：未区分正向/负向偏差，单次高奖励波动仍会引入大方差，性能提升有限；完善方向：正负偏差分离加权损失。

### 3.2 Phase2：环境迁移学习测试（未完善，不建议继续推进）
1. 实验目标：验证afterstate在环境偏移后，能否更快恢复原有得分水平；
2. 现存缺陷：
   - 无统一性能评判基准；困难环境下模型甚至无法恢复原始得分；
   - 训练对局数量受限，模型难以收敛至性能极限；
   - 单次完整训练耗时极长，实验数据参考价值低；

### 3.3 长期优化思路（未落地）
afterstate天然解耦玩家动作与环境随机过程；现有网络仅适配固定方块生成概率，后续可在白盒环境下，基于已知概率对同类afterstate局面做批量统一期望优化，提升跨环境泛化能力。

# 三、全文核心总结
## 3.1 afterstate 核心固有优势
- 解耦玩家确定动作与环境随机噪声，局面分布稳定、估值噪声低；
- 多路径搜索存在大量重复局面，可通过哈希缓存降低计算量；
- 天然适配V值学习，存储与训练成本低于Q网络；
- 高算力、高精度价值函数（N-tuple/深度网络）场景下性能优势完全释放。

## 3.2 afterstate 固有短板
- 估值内置固定环境随机概率假设，环境偏移时鲁棒性弱于原生state；
- 优势高度依赖高质量价值评估函数，简陋人工估值无法拉开性能差距；
- 低算力、少采样场景下稳定优势无法体现。

## 3.3 三类算法统一结论
1. Expectimax：afterstate提速效果最显著，搭配N-tuple可兼顾速度与得分；
2. MCTS：仅高算力+优质估值下afterstate占优，环境偏移衰减更严重；
3. 深度RL：afterstate更适配V值学习，但跨环境迁移效果仍存在明显缺陷。

总体分数结果基本符合预期，并且对于state而言，afterstate的时间消耗具有大幅度的下降趋势，因此考虑有关afterstate对于时间提升的机理：
1. 是通过只评判一个局面，而不是state取多种可能局面的均值（时间方面）
2. 并且afterstate得到的局面具有稳定性，其对应的Ntuple也具有稳定性，减少了噪音（环境方面）

因此可以得到：afterstate加上Ntuple是能够得到特定节点的具体值的，并且由于其具有极大的稳定性且噪音较少，因此对于节点的判断更为可靠，因此考虑可以尝试使用剪枝。

但真正实施的时候，由于本身的expectimax问题，剪枝无法完美实现，但是可以有一些替换：
1. 由于其得到的局面评判具有更高的稳定性——>我们考虑评判的结果是绝对正确的可能——>考虑使用类似beam search的方式，每轮次选择top2高的动作的局面，然后向下传递。最后的结果略高于我们不使用的方式——>时间缩减，并且深度越大缩减时间越明显，说明具有一定效果。但是针对于人工函数进行尝试，尽管仍旧具有时间缩减的效果，但是最后的得分降低很多。应当是由于人工函数难以真正评判真实的局面，从而让expectimax的“自信”招致毁灭。（已完成部分）
2. afterstate本身的局面是具有稳定性——>他在向下延申树枝的时候，多种局面会被重合，因此可以通过哈希表储存对应的局面以及对应的值，可以直接储存的值，无需计算（或许可以理解为从树变为有向无环图）（可比较轻易完成）

由于afterstate本身的平稳性，并且能够将环境和动作区别开的属性，我们认为其在环境困难或者环境有所误差的情况下具有更为良好的表现（phase2）

因此进行尝试：在先验与环境相同或者不同的情况下，不同环境下的结果的区别。

对于使用人工价值的函数的结果：（已完成，在phase2）
仍旧相差无几，并且得分都缓慢下降，没有代表性，因此不进行过多的考虑。

对于使用Ntuple部分的结果：（已完成，在phase2）
发现在对于afterstate的方式下，环境偏移导致的分数下降极为严重，甚至会导致比state下降差——>估计原因来源于对于局面的估计，afterstate得到的ntuple是隐含了10%概率的评估函数，因此评估概率的不准确会使得分数大幅下降；而state本身会估算所有的可能，第一是会严格考虑到恶意环境的可能，第二是其本就具有极大的噪声，因此会导致分数下降较为缓慢。（猜想）

后续方向：（部分，在phase_3）

因此考虑是否能够将局面的评估与风险评估分离，从而能够得到单纯的盘面估计以及关于概率可能的风险估计的双重Ntuple，从而让afterstate在不同环境中仍能保持相对的领先。因此后续也可以将风险估计用网络拟合为与概率相关的评估函数，从而让afterstate在不知道概率的黑盒2048环境中可以通过过往概率对其进行自适应判断。（不准备完成）

### 确立在mcts上只使用人工价值函数得到的结果，并进行不同算力导致的比拼。（phase1）

对于算力结果：在低算力情况下，在同样使用人工价值函数的情况下，afterstate比state会略显差，而在高算力情况下，afterstate比state会有明显提高。（已完成，在phase1）

首先，在只是人工价值函数的情况下，能够获得远超searching的分数，证明mcts算法本身具有极高的可取性。

其中phase1的方式已经经过改善：通过人工价值函数进行局面截断，避免2048的游戏方式让游戏时长过多（由于无上限，死亡才会回退，中间噪声过多），并使用rollout进行随机步数更新，确保对局面判断的稳定性。当算力较少的时候，两者没有明显的差别，原因是得到的反馈过少，很有可能具有危险的局面没有充分考虑到，或者对于某种局面没有充分考虑到其危险性。比如一旦生成4就会死亡，但是生成4的概率只有10%，而在200的rollout下，无法一定检测出来这样情况。

因此，在高算力的情况下，能够更为充分考虑到概率较小的可能性，会更容易将某盘面的价值收敛。并且由于其都是环境随机生成，afterstate本身在expectimax中可以缩减时间的能力在此处并不会进行显现，而对于深度，由于我们设置的随机行进，因此afterstate本身在这里提供的优势是对于动作返回值稳定的优势。

鉴于与expectimax同样的考量，也进行了环境偏移的测试方式，结果仍旧是都大幅下降，而afterstate下降的程度会更高，而且算力越高，在环境偏移后下降的幅度会越大。在同是人工价值函数的情况，我们认为这无法得到有效的结论，mcts本身具有极大的随机性，并且实验轮数较少，无法得到有效结果。（已完成，在phase2）

考量到afterstate本身，对于mcts中提供的优势是对于动作返回值的稳定性，因此我们具有的优势会是：对于单个局面我们具有更大的置信度。

由此，我们考虑更改mcts对于不同局面的选择方式，从先前的完全随机选择，变为由贪心和探索的组合方式，借由返回值对不同阶段的不同动作赋予价值，以此决定下一次的算力更倾向于往哪一条路探索。（完成一部分，可能有问题，在phase2的mcts）

得到的结果是对于人工价值函数而言，需要比原本更高的算力才能让afterstate比state得分更高——这是可以接受的解释，因为我们之前认为，afterstate本身具有一定的良好的性质，但是其缺陷是，只有当我们使用真正具有说明力的价值函数才能更好发挥能力。并且由于人工函数本身具有不完美性，确信这一点导致mcts的先验出现问题是正常的事情。

因此，后续希望对于mcts的价值函数转变为得到的Ntuple函数来进行进一步的判断，并希望能够得到更高的分数。于此同时，希望能够检测mcts可能进行的最大的分支，由于其具有更高的置信度，其最后形成的树应当是会更加瘦且深的，能够专注于更深层的价值。（有些类似）（未完成，在phase2中的mcts1）

而后未来的期望是，也类似expectimax的方式，能够真正判断局面和风险的价值，从而能够开辟真正有效的mcts的方位。（未完成，不准备完成）

### 确立在深度学习中使用不同的Q和V价值学习，分别针对于afterstate和state来进行探讨（phase1）

afterstate本身优势：能够直接学习V价值，而state本身没有这个能力。因此对于state而言，使用V会非常差劲。而对于Q而言，需要储存的东西远大于V。（已完成，在phase1）

考虑到最后的图像以及bias，其误差过大，其中可能是会具有训练局数不足的问题，但是我们训练太过困难，不希望太多的训练的局数，因此我们考虑产生MV，通过让其避免负向偏差值大的（具体来说，我们计算方差只考虑低于均值的误差，或者我们可以将其分开看待并分别辅以系数进行），由此，我们可以让其倾向于更稳健的策略，对将要走的路具有更为合理的预估。（MV的出现的预期）

但是由于没有考虑到可能会因为某次行动产生较大的奖励，导致方差过大的原因，因此提升效果不明显，如果能够将偏差分为正向偏差或者负向偏差，应当会更好。（或许可以对MV进行补全的地方）

而基于和之前同样的考虑，对其尝试实现了基于环境的迁移学习，我们希望能够看到我们的afterstate能够更快的回到与之前环境相同的得分——目前实验并不完善，并且此实验没有完美的标准——如果回到相同的得分是在更困难的环境中更为艰难的，甚至可能回不去。由于局数限制，我们无法通过尽可能多的局数让其真正收敛（比如达到其极限数字），也因此，没有很好的基准来进行判断。因此目前得到的结果具有极大的欠缺，并且运行时间很长，不建议继续完成。（完成部分，在phase2）

后续想法，afterstate好的一点是能够将玩家的动作与环境完美拆分开来，但是在目前的训练中仍旧没有达到更好的成绩。因此考虑是否能够通过将进行解耦后的局面，由原本的只能对于某种环境生成方式进行计算的方式，在白盒中根据概率来针对性对某个局面集体进行考量？（具体见Final_中层.md）
