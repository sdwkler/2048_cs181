# 2048 状态建模 (State vs Afterstate) Phase 2 研究逻辑溯源与总结

## 〇、前情提要与 Phase 2 核心动机
- **我们的起点 (Phase 1 结论)**：在第一阶段的基准测试中，我们发现 `afterstate` 展现出了极大的优势。在 Expectimax 中它能大幅缩减时间，在 MCTS 中它能提供更稳定的动作返回值，在强化学习中它能完美契合 $V(S')$ 网络的学习。
- **我们发现的潜在问题**：`afterstate` 的核心思想是将“玩家动作”与“环境随机发牌”解耦，只评估中间局面。
- **我们的猜想与方向**：既然 `afterstate` 将环境隔离了，那么当环境发生变化（例如生成数字 4 的概率 $P(4)$ 从标准的 10% 发生偏移、甚至变得极度恶劣时），`afterstate` 的表现是否会比必须硬算所有随机分支的 `state` 更有鲁棒性？
- **Phase 2 核心行动**：在 Expectimax、MCTS、Q-Learning 三个算法框架下，人为注入环境偏移（$P(4)$ 概率从 0.1 渐变到 0.9），开展压力测试。

---

## 一、Expectimax 模块：意料之外的脆弱

- **我们的方向**：在纯搜树框架下，验证 `afterstate` 估值函数在未知恶劣环境下的表现。
- **我们做了什么**：
  - 设计了 $P(4) \in [0.1, 0.9]$ 的 9 组梯度漂移环境。
  - 对比了 M1 (State+人工)、M2 (After+人工)、M3 (State+泛化 N-Tuple)、M6 (After+泛化 N-Tuple)。
  - 引入了 `M-Regret` 追踪：计算在高危环境下，State 与 Afterstate 决策分歧的概率（Disagreement Rate）和遗憾值。
- **得到了怎样的结果 (参考 expectimax_drift_full)**：
  - **断崖式下跌**：随着 $P(4)$ 升高，M6 (After+NT) 的得分从 113,551 暴跌至 19,356；而 M3 (State+NT) 虽然也从 64,406 跌至 11,458，但 M6 的绝对优势被严重抹平。
  - **决策分歧加剧**：环境越恶劣，Disagreement Rate 越高（从 0.26 升至 0.34），Afterstate 错判导致的 `max_regret` 飙升至 19819。
- **得到了怎样的结论**：
  - 我们的**初始猜想被推翻**。`afterstate` 的鲁棒性实际上**弱于**原生 `state`。
  - **内在机理**：`afterstate` 的 N-Tuple 估值本质上“暗含”了 10% 概率生成 4 的先验假设（过拟合了标准环境）。当环境偏移时，它的期望估值完全失真；而 `state` 每次决策都会全宽展开（Full-width expansion），天然地“算出”了当前环境的风险，反而具备抗噪能力。
- **面临的问题与下一步思考**：
  - 既然 `afterstate` 把环境概率“焊死”在了权重里，我们能否将“盘面价值”与“随机风险”拆分开？
  - **衍生（Phase 3 展望）**：提出双 N-Tuple 解耦设计（纯盘面网络 + 风险评估网络），让模型能自适应未知环境。

---

## 二、MCTS 模块：先验崩塌与信息熵上升

- **我们的方向**：MCTS 本身具备极强的随机采样抗噪能力，探索其在引入 UCB（贪心+探索）机制后，能否在恶劣环境下救场。
- **我们做了什么**：
  - 同样在 $P(4) \in [0.1, 0.9]$ 下测试 `MCTS-State` 与 `MCTS-After`。
  - 在底层引擎（`mcts3_node1.py`）中监控了树拓扑微观指标：`micro_variance`（动作区分度）、`micro_entropy`（访问混乱度/信息熵）、`micro_depth`（树深）。
- **得到了怎样的结果 (参考 mcts_drift_robustness_full)**：
  - **得分同步衰减**：`MCTS-After` 从 35,064 降至 15,410；`MCTS-State` 从 28,934 降至 13,976。两者均未展现出绝对的抗偏移神迹。
  - **微观指标异动**：随着环境变差，`micro_entropy` 居高不下。这说明在极高随机性的干扰下，MCTS 算力分配开始“雨露均沾”，难以锁定绝对优势分支。
- **得到了怎样的结论**：
  - 即使是动态采样的 MCTS，也无法完全抵消人工价值函数（Heuristic）在恶劣环境下的先验偏差。
  - `afterstate` 在 MCTS 中仍然需要比 `state` 更高的算力（更多的 Rollout）才能在环境偏移时修正估值误差。
- **面临的问题与下一步思考**：
  - 纯靠人工价值函数作截断，上限已被锁死，且环境适应性差。
  - **衍生（Phase 3 展望）**：考虑在 MCTS 中全面换用 N-Tuple，并深入剖析 `afterstate` 高置信度下形成的“瘦且深”的树拓扑结构，确保算力绝对收敛于正确分支。

---

## 三、Q-Learning 模块：零样本泛化的困境

- **我们的方向**：深度强化学习模型在标准环境训练收敛后，直接丢进漂移环境，测试其零样本 (Zero-Shot) 和连续学习 (Continual Learning) 的恢复能力。
- **我们做了什么**：
  - 设计实验 3-A 到 3-E，特别是对比 `V(s') + Afterstate` (3-D) 和加入了下行方差约束的 `MV(s')` (3-E)。
  - 进行 Zero-Shot 测试，观察模型在无微调情况下的生存能力。
- **得到了怎样的结果 (参考 qlearning_drift_results_smoke)**：
  - **全面崩溃**：最优的 3-D 模型，在 $P(4)=0.1$ 时得分为 19,767，但当 $P(4)=0.9$ 时，暴跌至 5,241。加入稳健性约束的 3-E 模型（17,957 $\rightarrow$ 5,140）也未能幸免。
  - **迁移学习失败**：试图在困难环境中继续训练让其恢复分数的实验（Continual Learning）耗时极长，且往往因为陷入次优解而无法收敛回原水平（负迁移现象）。
- **得到了怎样的结论**：
  - 强化学习拟合出的 V 网络同样将标准环境的白噪声当成了固有规律。
  - 时序差分（TD）目标在单步采样下，被恶劣环境的方差彻底撕裂，模型“吓坏了”，策略退化为极度保守或彻底混乱。
- **面临的问题与下一步思考**：
  - 单步采样计算 TD Target 无法应对环境的高方差。
  - **衍生（Phase 3 展望）**：抛弃单步采样，利用已知白盒概率进行 **全宽期望目标 (TDA-Full)** 优化（`run_qlearning_new3.py` 中实现的 3-E/3-F），强制让网络拟合数学期望而非单次采样的奖励，并完善正负偏差分离的 `Downside-MV` 损失函数。

---

## 四、全链路核心总结

贯穿 Phase 2 的测试，我们得出了一个违背直觉但极具价值的结论：

**解耦环境 $\neq$ 免疫环境。**

`afterstate` 通过状态截断，确实在计算效率（Phase 1）上取得了巨大成功。但在状态建模的过程中，它不可避免地将环境的**概率分布**吸纳进了自身的价值函数中。
1. 当环境是**静态且符合预期**时，它是降维打击的利器（低噪、高效）。
2. 当环境是**动态或发生漂移**时，它是作茧自缚的枷锁（刻板偏见、错估风险）。

这也直接指引了我们后续的研究终局：要想真正发挥 `afterstate` 的威力，必须从**更新目标（期望 TD）**和**架构解耦（风险网络与价值网络双头机制）**上进行底层重构。







# 2048 状态建模 (State vs Afterstate) Phase 3 演进逻辑与深度总结

## 〇、前情提要与 Phase 3 核心动机
- **我们的起点 (Phase 2 遗留痛点)**：在环境漂移测试中，我们发现 `afterstate` 虽然提速明显，但它将环境概率（$P(4)$）“隐式过拟合”到了价值网络中，导致鲁棒性崩塌。
- **理想中的完美解法**：“双 N-Tuple 解耦”——训练一套纯评估盘面的网络，再训练一套专门评估环境随机风险的网络，运行时动态结合。
- **现实的妥协与战略转移**：为什么我们**放弃了完全解耦**？
  1. **工程灾难与算力限制**：要让 N-Tuple 动态适应不断变化的白盒概率，需要让特征空间维度爆炸，彻底打破 N-Tuple 查表带来的 $O(1)$ 极速优势。
  2. **思路转换**：与其强行拆解 N-Tuple，不如“退而求其次”，发挥 `afterstate` 最底层的物理特性——**动作确定性**与**高置信度**。我们决定从“树的搜索结构优化”和“强化学习的目标函数重构”两个维度，去弥补它在环境漂移中的缺陷。

---

## 一、Expectimax 模块：从解耦的幻想，到数据结构的降维打击

- **我们的思考与方向**：
  既然不能修改 N-Tuple 的内核，我们就榨干 `afterstate` 纯确定性局面的红利。在穷举树中，不同的动作组合（如“上下上”和“上上下”）极大概率会收敛到同一个盘面。`state` 包含了随机生成的方块，很难重合；但 `afterstate` 仅仅是玩家操作后的盘面，**重合率极高**。此外，既然 `afterstate` 结合 N-Tuple 估值极其准确，我们能否提前砍掉那些看起来就很蠢的分支？
- **我们做了什么**：
  1. **HashDAG 缓存去重 (无损优化)**：利用哈希表记忆已搜索过的 `afterstate` 节点，将传统的树（Tree）结构折叠为有向无环图（DAG）。
  2. **Top-2 Beam Search 剪枝 (有损加速)**：基于 `afterstate` 的高置信度估值，每一步仅保留得分最高的两个动作向下展开，强行降低搜索分支因子。
- **得到了怎样的结果 (基于 `search_optimizations_full` 结论)**：
  - **DAG 优化**：得分与 Base 完全一致（完美对齐），但在深层搜索时，展开节点数呈指数级下降，时间压缩比（Compression Ratio）极高。
  - **剪枝优化**：以极小（约 10%）的分数折损，换取了夸张的速度提升（尤其在深层搜索时）。但我们同时发现，如果退化使用人工价值函数（Heuristic），剪枝会导致分数大雪崩。
- **得到了怎样的结论**：
  - **高精度估值是剪枝的灵魂**。人工函数的“盲目自信”会剪掉生路，而 N-Tuple 赋予了 `afterstate` 真正的判断力。
  - `afterstate` 天然是 DAG 数据结构的最佳拍档，它在纯搜索算法中的核心定位不仅是“少算随机分支”，更是“大幅提升状态复用率”。

---

## 二、MCTS 模块：拨开随机迷雾，看见“瘦且深”的拓扑真容

- **我们的思考与方向**：
  在 Phase 2 中，我们发现 MCTS 在恶劣环境下信息熵极高（到处乱搜）。既然 Phase 3 我们确认了 `afterstate` 结合 N-Tuple 具备极高置信度，那么如果把 N-Tuple 接入 MCTS，MCTS 的探索树结构会发生什么物理形态上的改变？
- **我们做了什么**：
  - 设计微观拓扑追踪实验（`run_topology_experiment.py`）。
  - 在“绝境求生”（盘面极度受限）的态势下，剖析 `state` 和 `afterstate` 的算力分配比例，提取 `micro_entropy`（访问混乱度）和 `micro_depth`（树深）。
- **得到了怎样的结果 (基于 `mcts_topology_analysis_full`)**：
  - 在死局边缘，`state` 树的形态像一把“矮胖的伞”，因为环境的高方差让算力被迫分散在各个可能性上，信息熵飙升。
  - `afterstate` 树的形态像一把“锐利的长剑”（极深且瘦）。由于中间局面的回报方差极小，UCB 算法迅速收敛，将 $90\%+$ 的算力绝对聚焦在唯一正确的求生分支上，向下挖掘极深。
- **得到了怎样的结论**：
  - `afterstate` 在 MCTS 中的终极意义被揭示：它通过**降噪**，欺骗了 UCB 公式中的探索项，强行将 MCTS 从“广度优先采样”扭转成了“深度优先规划”。这解释了为什么它能规避低级随机死亡。

---

## 三、Q-Learning 模块：用数学期望重塑 TD 目标 (基于当前代码设计的实验预测)

- **我们的思考与方向**：
  回到 Phase 2 的核心痛点——环境漂移导致单步采样（Single-step Sample）计算出的 TD Target 方差过大。既然 2048 是一个白盒环境（我们知道具体哪些格子是空的），我们为什么还要靠“扔骰子（采样）”来更新网络？
  直接利用 `afterstate` 的特性，遍历所有可能的生成位置，计算一个**绝对正确的数学期望**，不就能从根源上消除环境白噪声了吗？
- **我们做了什么 (即代码中 3-E / 3-F / 3-G 的设计)**：
  1. **TDA-Full (全宽期望目标)**：在计算目标值时，公式从 $r + \gamma V(S'_{next})$ 变更为对所有可能的环境发牌结果求期望：$r + \gamma \sum P(e) V(S'_{next}|e)$。
  2. **Downside-MV (下行方差约束重构)**：修正了原来的方差设计。我们意识到“正向偏差”（突然合成高阶方块得到的极高分）对模型是奖励，不该被惩罚！我们只对“低于期望的负向偏差（下行风险）”施加 Loss 惩罚。
- **我们预测会得到怎样的结果 (预期现象)**：
  - **TDA-Full 的降维打击**：由于完全抹除了环境采样的白噪声，TD-Error 的收敛曲线将变得极其平滑（不会再出现 Phase 2 中断崖式的 Error 冲击）。零样本（Zero-Shot）泛化能力会有肉眼可见的回升，因为模型学到的是“期望价值”而非“采样特例”。
  - **Downside-MV 的策略质变**：模型展现出“贪心且保守”的非对称行为。在安全时，它会激进地追求正向收益；在危险时，它会对下行风险产生强烈的排斥，生存步数（Survival Steps）将显著提升。
- **得到了怎样的终极结论**：
  - 解耦环境不需要真的去建两个网络，**将环境的先验概率直接融入强化学习的数学目标（TD Target）中**，是更为优雅、计算更高效的“数学级解耦”。
  - `afterstate` 配合 TDA-Full，才是 2048 强化学习能够抵御恶劣环境的终极形态。
