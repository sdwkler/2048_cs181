🔬 III. Two Separable Dimensions & 2×2 Ablation Matrix

Dimension 1: Tree Structure

Controls whether the search tree explicitly inserts S' chance nodes between action nodes and stochastic tile spawns.

No separation: After each action, environment immediately spawns a random tile, producing N different S_next states → tree explodes at chance nodes.
Separation: After each action, first reach S' (merge result only); different actions may produce identical S' → search tree collapses to DAG.

Effect Compression ratio 1.0→0.55, 10.3× speedup

Dimension 2: Evaluation Target

Controls whether the evaluator sees clean S' or noisy S_next.

Evaluate S_next: Leaf nodes must simulate stochastic tile spawn → noise contaminates value estimates.
Evaluate S': Direct evaluation of clean board — but evaluator must be trained on matching S' distribution.

Effect Score +77% (only when evaluator input distribution matches)

💡 VIII. Core Findings

Finding 1: Joint Decoupling is Necessary

Two dimensions cannot be used separately; mismatch cost is highly asymmetric.

Expectimax: 1-E (evaluator-only, mismatched) collapses to 5,066 (-92%); reverse 1-F (tree-only) drops only -7%.

1-G full decoupling: +77% score + 10.3x speedup — both dimensions together unlock the full benefit.

Finding 2: Gain Decays with Evaluator Precision

Afterstate is an amplifier, not a creator of precision.

Pre-trained N-Tuple (Expectimax +77%) → Online TD 25K (3-D +431% vs 3-A) → Pure Rollout MCTS (only +12%).

MCTS at low budget (200/500) even loses to State — noise isolation below threshold.

Finding 3: Overestimation is Structural

ε+α annealing only delays, does not cure.

25K: norm_bias_rom = -0.194 (slight underestimation)
100K: norm_bias_rom = +1.976 (2x overestimation)

Deadly Triad: N-Tuple capacity ceiling + Bootstrapping + Off-policy → positive feedback. All QL variants plateau at ~18-20K.

Conclusion: Afterstate decoupling is effective under standard conditions (Expectimax +77% -90% time; TD +431%), but requires three strict preconditions: (1) both dimensions must be used jointly, (2) evaluator precision must be sufficient, (3) environment probabilities must remain stable. Under environment drift, Afterstate is less robust than State. Overestimation structurally regresses in long-term TD training. Two carefully designed architecture improvements (TDA-Full, Downside-MV) both failed to beat the simple baseline after sufficient training. MCTS+N-Tuple (136K) surpassing Expectimax (115K) reveals that evaluator precision changes the optimal algorithm choice.
